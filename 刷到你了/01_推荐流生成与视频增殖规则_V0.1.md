# 《刷到你了》推荐流生成与视频增殖规则 V0.1

> 文档性质：核心玩法专项规则  
> 对应总纲：《00_初步头脑风暴.md》  
> 当前范围：视频链接、商品购买、物品打赏、后续视频生成、信息流扩容与整理

## 一、规则目标

本系统负责实现游戏最核心的体验：

> 玩家从少量看似无关的视频开始，通过购买和跨视频打赏不断生成新视频，使推荐流从三条内容逐渐生长成一张可以探索、解谜和整理的因果网络。

推荐流的增长本身就是进度反馈。玩家不是进入下一张传统地图，而是亲眼看着一个原本空荡的信息流被自己的操作填满。

## 二、核心定义

### 2.1 视频节点

推荐流中的每一条短视频、直播回放、广告视频和反应视频，都是一个独立的“视频节点”。

每个视频节点至少记录：

- 视频编号；
- 内容类型；
- 发布账号；
- 视频发生的时间；
- 当前是否出现在推荐流；
- 可点击的商品链接；
- 可以接收的礼物；
- 已经收到的礼物；
- 可以生成的后续视频；
- 它由哪条视频和哪次操作触发。

### 2.2 原视频与后续视频

原视频是玩家执行打赏操作的目标，后续视频是这次操作产生的新内容。

例如：

- B 视频是某位主人公发布的原视频；
- 玩家把 X 礼物打赏进 B 视频；
- 系统生成 C 视频；
- C 视频表现 B 视频中的主人公收到 X 后的反应或后续经历。

**B 不会被 C 替换。**

B 是因果起点和历史记录，C 是它产生的新结果。两条视频会同时存在于推荐流中。玩家可以回看 B，理解 C 为什么出现，也可以继续对 B 或 C 进行新的操作。

### 2.3 当前时间线

首版采用单一累积时间线，不默认使用多重宇宙。

新生成的视频是当前世界发生的新后续，而不是与旧结果互斥的平行结局。原视频仍然存在，是因为短视频内容本身是一份已经发布的记录；世界状态则会继续向前累积。

如果玩家之后把另一件礼物 Y 送进 B，系统应读取 B 已经收到过 X 的状态，再决定生成“收到 X 和 Y 后”的新后续，而不是假装 X 从未出现。

需要体验互斥解法时，玩家通过关卡回退或重置重新选择，不让互相冲突的世界状态同时成为当前事实。

## 三、关卡初始状态

### 3.1 初始视频数量

一关开始时，推荐流中可以只有 **3 条视频**。

这三条视频需要分别承担不同功能：

1. **商品来源**：包含可以点击的商品链接；
2. **潜在目标**：允许玩家把商品打赏进去，并生成第一条后续视频；
3. **背景线索或玩梗支线**：帮助玩家理解人物关系，也可以承接非标准操作。

三条内容不直接标注“起点”“目标”或“线索”。玩家通过反复刷动、观察评论和点击链接，自行发现第一步操作。

### 3.2 初始物品状态

玩家开局时：

- 礼物背包为空；
- 已拥有礼物数量为 0；
- 商城不直接展示完整关卡商品；
- 只有视频中已经出现的挂链商品可以购买。

首个教学商品 X 必须保证玩家能够购买。具体可以采用免费试用、教学赠币或足额初始金币，但不能让经济系统阻断第一次核心操作。

### 3.3 初始引导原则

教学不直接显示“请购买 X 并送给 B”。推荐做法是：

- 第一次刷到商品链接时，链接出现轻微动效；
- 点击链接后展示“购买后可作为礼物使用”；
- 购买完成后，第一次打开礼物栏时提示“选择一条视频送入”；
- B 视频通过画面需求、人物台词或评论暗示它可能需要 X。

教学只解释操作方式，不直接公布谜题答案。

## 四、商品链接与购买

### 4.1 链接出现位置

商品链接可以出现在：

- 直播间的小黄车；
- 普通视频下方的挂链；
- 广告视频的商品按钮；
- 评论区置顶链接；
- 后续视频中新解锁的关联商品。

### 4.2 点击链接

点击链接后，以覆盖层打开简化商品页，不把玩家带离当前关卡。

商品页至少显示：

- 商品名称和外观；
- 价格；
- 一句带有玩梗或线索性质的商品描述；
- 当前持有数量；
- 购买按钮。

打开商品页时暂停需要玩家即时判断的直播进程，避免玩家因为阅读商品说明而错过关键时机。关闭商品页后，直播从暂停点继续。

### 4.3 购买结果

购买成功后：

- 商品 X 进入礼物背包；
- 商品链接保持可访问；
- 玩家回到原视频的同一观看位置；
- 礼物栏出现一次短促提示；
- 不自动指定投放目标。

商品一旦被某条视频解锁，即使玩家之后隐藏该视频，商品仍保留在商城的“已发现”分类中，避免隐藏视频导致关卡软锁。

## 五、物品打赏

### 5.1 发起打赏

玩家在任意支持打赏的视频中打开礼物栏，选择已购买的物品，再确认投放。

打赏需要清楚显示三个信息：

- 当前目标视频；
- 即将投入的物品；
- 投入后物品是否消耗。

V0.1 暂按“一次购买对应一次投放”设计。关卡重置后恢复教学必需资源；物品是否允许批量购买和回收，在经济规则文档中另行确定。

### 5.2 投放动画

确认后执行统一反馈：

1. 当前视频短暂停顿；
2. 礼物以直播打赏特效进入画面；
3. 礼物特效从二维界面转为视频世界中的真实物品；
4. 视频人物注意到、捡起或使用物品；
5. 系统记录“目标视频 + 礼物”的因果触发；
6. 进入后续视频生成阶段。

动画需要短、清晰、有冲击力，目标时长控制在 1～3 秒，避免破坏刷视频的节奏。

### 5.3 无效与非标准投放

首版不鼓励出现纯粹的“无效礼物”。物品投错时优先产生以下结果之一：

- 一条短小的搞笑反应；
- 评论或弹幕变化；
- 新广告；
- 物品被人物转送给其他人；
- 一条不推进主线但可继续互动的支线视频。

同一件物品对同一视频的完全相同触发只结算一次。玩家重复执行时，界面提示“这份礼物已经送过”，不重复扣除物品，也不生成重复视频。

## 六、后续视频生成

### 6.1 生成条件

系统根据以下信息选择新视频：

- 目标视频；
- 投入物品；
- 目标人物当前状态；
- 该人物此前收到过的礼物；
- 当前已完成的主线和支线事件；
- 是否满足特殊组合条件。

生成不是依赖实时 AI 随机创作，而是从关卡预先设计好的视频节点与触发条件中选择，确保谜题逻辑和喜剧节奏可控。

### 6.2 即时播放

当 B + X 成功触发 C 时：

1. B 视频完成打赏反馈；
2. 界面出现“为你找到一条新视频”或同类平台提示；
3. C 视频立即进入全屏播放；
4. C 展现 B 中主人公收到 X 后的反应；
5. 播放结束后，玩家可以继续向上刷动；
6. B 与 C 都保留在推荐流中。

即时播放保证玩家立刻看到行为结果，是核心正反馈，不能把 C 静默塞到信息流深处。

### 6.3 新节点插入位置

C 首次生成时立即播放；播放后进入正常推荐流。

推荐流排序遵循两层规则：

- 新生成内容在短时间内具有最高推荐权重；
- 当新鲜权重结束后，C 依照人物、时间和因果关联参与正常排序。

在 B 的详情页或关联入口中，应能看到“由此视频产生的后续”，方便玩家回顾 B → C 的关系。主推荐流不强制把 B 和 C 永远相邻，以保留刷视频的自然感。

### 6.4 视频数量增长

初始推荐流有 3 条视频。每触发一条新的有效后续，推荐流总量增加 1。

示例：

| 阶段 | 推荐流内容 | 数量 |
| --- | --- | ---: |
| 初始 | A、B、D | 3 |
| 从 A 购买 X，打赏给 B | A、B、C、D | 4 |
| C 解锁广告 E | A、B、C、D、E | 5 |
| 把 E 中的 Y 打赏给 D，生成 F | A、B、C、D、E、F | 6 |

旧视频不会因为新视频出现而自动消失。玩家看到的信息流会随着探索逐渐膨胀，这种膨胀就是关卡探索范围的可视化。

## 七、因果链与分支

### 7.1 因果关系

每条新视频都必须能回答三个问题：

1. 它由哪条视频触发？
2. 玩家送入了什么物品？
3. 它改变了谁的状态或解锁了什么内容？

如果一条视频无法清楚回答这三个问题，就不应作为物品投放的直接结果生成。

### 7.2 主线增长

主线视频负责：

- 推进核心谜题；
- 解锁关键商品；
- 改变主要人物状态；
- 生成关卡收束所需的内容。

### 7.3 支线增长

支线视频负责：

- 回应非标准投放；
- 制造搞笑结果；
- 补充人物关系；
- 提供额外资源或替代路线；
- 让玩家感觉自己的尝试被世界认真回应。

支线可以继续生成支线，但需要控制深度。建议普通玩梗支线最多连续生长 2～3 层，避免一条笑话无限扩张并淹没核心谜题。

### 7.4 组合反应

同一人物先后收到不同礼物时，可以触发组合视频。

例如：

- 单独送梯子，人物爬上屋顶；
- 单独送扩音器，人物在楼下喊话；
- 已经送过梯子后再送扩音器，生成“站在屋顶公开喊话”的特殊视频。

组合反应是高价值隐藏内容，适合奖励主动试验的玩家，不要求每一对物品都有组合结果。

## 八、推荐流超过 15 条后的整理

### 8.1 整理功能解锁

当当前关卡累计生成第 16 条视频时，解锁“不感兴趣”功能，并进行一次轻量提示：

> 推荐内容有点多？可以隐藏暂时不需要的视频。

15 条不是推荐流的硬上限。玩家可以继续保留更多内容，也可以主动整理。

### 8.2 不感兴趣

玩家点击视频菜单中的“不感兴趣”后：

- 该视频从当前推荐流中隐藏；
- 推荐算法降低同类内容的短期权重；
- 视频已经造成的世界变化不会撤销；
- 由它解锁的商品、人物和后续视频不会消失；
- 该视频不会被永久删除。

这里的“删除”是信息流层面的隐藏，不是因果层面的抹除。否则玩家可能因为删除关键来源视频而破坏关卡逻辑。

### 8.3 已隐藏内容

玩家可以从“浏览记录 / 不感兴趣”列表找回被隐藏的视频。

恢复后：

- 视频重新参与推荐；
- 原有观看记录、礼物记录和因果关系全部保留；
- 不重复触发首次观看奖励。

### 8.4 收藏

为配合信息流整理，玩家从游戏开始就可以收藏视频。收藏不改变因果状态，只帮助玩家保留重要线索。

当玩家准备隐藏一条仍有未触发关键链接的视频时，系统只提示“该视频还有未查看内容”，但允许玩家继续隐藏。因为视频可以恢复，系统不需要禁止玩家操作。

## 九、防卡死规则

为了让信息流持续增长而不破坏解谜，系统必须满足以下规则：

1. 隐藏视频不会撤销世界状态；
2. 已解锁商品不会因来源视频隐藏而重新上锁；
3. 隐藏视频可以随时恢复；
4. 重复投放不会重复扣除物品；
5. 关键教学商品保证可购买；
6. 每关提供关卡回退或完整重置；
7. 当前时间线出现互斥状态时，以最近确认的关键选择为准，并提示玩家需要回退才能体验另一结果；
8. 关卡目标所需的唯一视频被隐藏时，目标提示可引导玩家查看“不感兴趣”列表，但不自动恢复。

## 十、完整教学示例

### 10.1 初始推荐流

- **A：带货直播间**——主播正在演示 X，直播间有 X 的购买链接；
- **B：主人公的普通视频**——画面或评论暗示主人公缺少 X；
- **D：无关搞笑视频**——表面与主线无关，但可以接受 X 并产生玩梗支线。

玩家开局没有任何礼物。

### 10.2 标准路线

1. 玩家刷到 A，点击直播间挂链；
2. 购买 X，X 进入礼物背包；
3. 玩家回到推荐流，刷到 B；
4. 打开礼物栏，把 X 打赏进 B；
5. X 从界面礼物变成 B 视频世界里的真实物品；
6. B 的主人公做出即时反应；
7. 系统生成并立即播放 C；
8. C 展示主人公收到 X 后的进一步结果；
9. 推荐流由 A、B、D 三条增长为 A、B、C、D 四条；
10. C 中出现新线索或新广告，推动下一轮购买与投放。

### 10.3 非标准路线

如果玩家把 X 送给 D：

- D 产生一条完整的搞笑反应视频 D1；
- 推荐流由 3 条增长到 4 条；
- 玩家得到支线反馈，系统同时保证玩家仍能通过再次购买、支线奖励或回退取得完成主线所需的 X；
- D1 可以提供少量资源、梗图或间接线索，但不代替主线 C。

非标准路线不能让玩家永久失去完成主谜题的机会。

## 十一、关卡设计数据要求

每设计一条可互动视频，需要同步填写：

- 视频 ID 与标题；
- 发布者与发生时间；
- 视频的表层笑点；
- 视频提供的线索；
- 挂载商品；
- 可接收物品清单；
- 每个物品对应的即时反应；
- 生成的后续视频 ID；
- 对人物状态的修改；
- 对商品和广告的解锁；
- 是否属于主线；
- 是否允许继续生长；
- 被隐藏后是否需要特殊提示。

每设计一件物品，需要同步填写：

- 物品 ID 与名称；
- 来源视频或商品链接；
- 价格与可购买次数；
- 标准用途；
- 主线目标视频；
- 可产生内容的支线目标；
- 特殊组合反应；
- 重复投放规则；
- 重置后的恢复规则。

## 十二、本版明确规则

- 每关可以从 3 条视频和 0 件礼物开始；
- 第一件礼物来自其中一条视频或直播的商品链接；
- 玩家购买礼物后自行判断投放目标；
- B 收到 X 后新增 C，B 本身继续保留；
- 视频以因果节点方式不断增长，不自动替换旧视频；
- 新视频首次生成时立即播放；
- 首版采用单一累积时间线；
- 同一触发组合只生成一次，不刷重复视频；
- 普通玩梗支线建议最多连续生长 2～3 层；
- 超过 15 条视频后解锁“不感兴趣”；
- “不感兴趣”只隐藏视频，不删除因果结果；
- 隐藏视频可以从记录中恢复；
- 已解锁商品不会因视频隐藏而丢失；
- 非标准投放也应产生内容，不能造成永久卡关。

## 十三、下一步应单独讨论的规则

以下内容不在本文中混写，后续分别形成专项文档：

1. 物品库存、金币、购买次数与回收规则；
2. 视频节点、人物状态与触发条件的数据表格式；
3. 关卡目标、结局判断、回退和重置规则；
4. 直播倒计时、暂停和连续打赏规则；
5. 第一关的完整视频树、物品表和解法表；
6. 推荐排序、收藏、搜索和账号主页规则。

# 《刷到你了》推荐流生成与视频增殖规则 V0.1

> **规则更新提示**：关于可接受物品的视频范围、资源树正确配对、植入链接和关键商品解锁方式，以《04_树状推荐流与语义物品设计规则_V0.1.md》为准。本文中“只有可改命失败视频能接受物品”的限制已被新规则放宽。

> 文档性质：核心玩法专项规则  
> 对应总纲：《00_初步头脑风暴.md》  
> 当前范围：视频链接、商品购买、物品打赏、预制结果视频生成、阶段结算、信息流扩容与整理

## 一、规则目标

本系统负责实现游戏最核心的体验：

> 玩家从少量看似无关的视频开始，通过购买和跨视频打赏不断生成新视频，使推荐流从三条内容逐渐生长成一张可以探索、解谜和整理的因果网络。

推荐流的增长本身就是进度反馈。玩家不是进入下一张传统地图，而是亲眼看着一个原本空荡的信息流被自己的操作填满。

## 二、核心定义

### 2.1 视频节点

推荐流中的每一条短视频、直播回放、广告视频和反应视频，都是一个独立的“视频节点”。

每个视频节点至少记录：

- 视频编号；
- 内容类型；
- 发布账号；
- 视频发生的时间；
- 当前是否出现在推荐流；
- 可点击的商品链接；
- 可以接收的礼物；
- 已经收到的礼物；
- 可以生成的后续视频；
- 它由哪条视频和哪次操作触发。

### 2.2 原视频与后续视频

原视频是玩家执行打赏操作的目标，后续视频是这次操作产生的新内容。

例如：

- B 视频是某位主人公发布的原视频；
- 玩家把 X 礼物打赏进 B 视频；
- 系统生成 C 视频；
- C 视频表现 B 视频中的主人公收到 X 后的反应或后续经历。

**普通触发时，B 不会被 C 替换。**

B 是因果起点和历史记录，C 是它产生的新结果。两条视频会同时存在于推荐流中。玩家可以回看 B，理解 C 为什么出现，也可以继续对 B 或 C 进行新的操作。

当 C 属于“阶段结果视频”时规则不同：已经被彻底解决的问题状态视频会从当前推荐流退场，进入历史记录。它们没有被删除，但在玩家回退阶段前不再接受礼物。阶段结果视频成为新的当前状态和下一阶段起点。

### 2.3 失败视频与逆天改命

主线中允许玩家投放礼物的目标必须是一条已经无法继续的失败视频。失败至少满足以下一种情况：

- 主角死亡；
- 当前核心事件彻底失败，例如婚礼取消；
- 主角进入与目标无关的另一条人生线，原目标不再可能达成。

玩家不是给正常进行的剧情添加便利，而是把礼物送回失败发生前的关键时刻，生成一条被改写后的新视频。

### 2.4 当前婚礼线与荒诞终局

首版同时记录多种结果，但推荐流中只追踪一条仍有机会完成目标的当前主线。

正确配对生成的结果进入当前主线；错误配对生成的死亡结局或其他人生线进入“命运记录”，但不替换当前主线状态。玩家看完荒诞终局后返回原失败视频继续改命。

主线按固定顺序累积正确条件。失败视频 B 只接受唯一正确礼物 X，生成仍然失败但保留 X 条件的新视频 C；C 再接受唯一正确礼物 Y，生成下一结果。Y 直接送进 B 属于错误配对，不能提前满足后续条件。

荒诞终局是玩家发现过的可能命运，不会与当前婚礼线同时成为事实。

## 三、关卡初始状态

### 3.1 初始视频数量

一关开始时，推荐流中可以只有 **3 条视频**。

这三条视频需要分别承担不同功能：

1. **主线失败视频**：以死亡或目标彻底失败收尾，是第一条可改命视频；
2. **第一商品来源**：包含可以点击的关键商品链接；
3. **第二商品来源或主线线索**：提供另一项正确条件，但本身不是可改命目标。

三条内容不直接标注“起点”“目标”或“线索”。玩家通过反复刷动、观察评论和点击链接，自行发现第一步操作。

### 3.2 初始物品状态

玩家开局时：

- 礼物背包为空；
- 已拥有礼物数量为 0；
- 商城不直接展示完整关卡商品；
- 只有视频中已经出现的挂链商品可以购买。

首个教学商品 X 必须保证玩家能够购买。具体可以采用免费试用、教学赠币或足额初始金币，但不能让经济系统阻断第一次核心操作。

### 3.3 初始引导原则

教学不直接显示“请购买 X 并送给 B”。推荐做法是：

- 第一次刷到商品链接时，链接出现轻微动效；
- 点击链接后展示“购买后可作为礼物使用”；
- 购买完成后，第一次打开礼物栏时提示“选择一条视频送入”；
- 主线失败视频通过死亡原因、画面线索和评论暗示它可能需要 X。

教学只解释操作方式，不直接公布谜题答案。

## 四、商品链接与购买

### 4.1 链接出现位置

商品链接可以出现在：

- 直播间的小黄车；
- 普通视频下方的挂链；
- 广告视频的商品按钮；
- 评论区置顶链接；
- 后续视频中新解锁的关联商品。

### 4.2 点击链接

点击链接后，以覆盖层打开简化商品页，不把玩家带离当前关卡。

商品页至少显示：

- 商品名称和外观；
- 价格；
- 一句带有玩梗或线索性质的商品描述；
- 当前持有数量；
- 购买按钮。

直播间底层仍是预制视频素材，不执行真正的实时演算。打开商品页时暂停直播视频播放，关闭商品页后从暂停点继续，避免玩家因为阅读商品说明而错过内容。

### 4.3 购买结果

购买成功后：

- 商品 X 进入礼物背包；
- 商品链接保持可访问；
- 玩家回到原视频的同一观看位置；
- 礼物栏出现一次短促提示；
- 不自动指定投放目标。

商品一旦被某条视频解锁，即使玩家之后隐藏该视频，商品仍保留在商城的“已发现”分类中，避免隐藏视频导致关卡软锁。

## 五、物品打赏

### 5.1 发起打赏

玩家只能在标记为“可改命”的失败视频中打开礼物栏，选择已购买的物品，再确认投放。正常进行中的剧情视频、商品来源和普通结果视频默认不能被改命。

打赏需要清楚显示三个信息：

- 当前目标视频；
- 即将投入的物品；
- 投入后物品是否消耗。

V0.1 暂按“一次购买对应一次投放”设计。关卡重置后恢复教学必需资源；物品是否允许批量购买和回收，在经济规则文档中另行确定。

### 5.2 投放动画

确认后执行统一反馈：

1. 当前视频短暂停顿；
2. 界面播放统一的直播打赏特效；
3. 原视频素材本身不新增人物表演，也不要求动态拼接；
4. 系统记录“目标视频 + 礼物”的因果触发；
5. 系统选中一条预先制作好的结果视频；
6. 礼物特效结束后立即全屏播放结果视频；
7. 人物在结果视频中收到、捡起或使用真实物品。

动画需要短、清晰、有冲击力，目标时长控制在 1～3 秒，避免破坏刷视频的节奏。

### 5.3 错误投放与荒诞分支

每个可互动主线视频只配置一个正确物品。把该物品投进该视频会生成下一个主线结果；其他所有配对都视为错误投放。

错误投放不显示“无效礼物”，而是生成一条荒诞结果视频，例如：

- 一条短小的搞笑反应；
- 评论或弹幕变化；
- 物品被人物转送给其他人；
- 一条不推进主线的夸张反应视频。

荒诞结果必须形成一个完整终局：主角死亡，或主角进入不再追求当前目标的另一条人生线。它不能修改当前主线状态、解锁主线商品或替代正确解法。播放结束后，原失败视频仍保持可改命状态。荒诞终局进入命运记录，不再接受礼物。

同一件物品对同一视频的完全相同触发只结算一次。玩家重复执行时，界面提示“这份礼物已经送过”，不重复扣除物品，也不生成重复视频。

## 六、后续视频生成

### 6.1 生成条件

系统根据以下信息选择新视频：

- 目标视频；
- 投入物品；
- 目标人物当前状态；
- 该人物此前收到过的礼物；
- 当前已完成的主线事件和已看过的荒诞分支；
- 是否满足特殊组合条件。

生成不是依赖实时 AI 随机创作，而是从关卡预先设计好的视频节点与触发条件中选择，确保谜题逻辑和喜剧节奏可控。

### 6.2 即时播放

当 B + X 成功触发 C 时：

1. B 视频暂停，播放通用礼物特效；
2. B 的原始素材不发生变化；
3. 特效结束后，C 视频立即进入全屏播放；
4. C 展现 B 中主人公收到 X 后的反应；
5. 播放结束后，玩家可以继续向上刷动；
6. 正确触发时 B 与 C 暂时都保留；阶段结算时，已经过时的 B 进入历史记录；
7. 错误触发时 C 作为荒诞终局进入命运记录，播放器返回仍未解决的 B。

即时播放保证玩家立刻看到行为结果，是核心正反馈，不能把 C 静默塞到信息流深处。

直播间使用相同流程。直播画面只承担信息、弹幕、商品链接与氛围，不在原素材内即时回应礼物。打赏直播后生成的 C 可以包装成直播切片、下播回应或主播刚发布的普通视频，但底层仍是预制视频节点。

### 6.3 新节点插入位置

C 首次生成时立即播放；播放后进入正常推荐流。

推荐流排序遵循两层规则：

- 新生成内容在短时间内具有最高推荐权重；
- 当新鲜权重结束后，C 依照人物、时间和因果关联参与正常排序。

在 B 的详情页或关联入口中，应能看到“由此视频产生的后续”，方便玩家回顾 B → C 的关系。主推荐流不强制把 B 和 C 永远相邻，以保留刷视频的自然感。

### 6.4 视频数量增长

初始推荐流有 3 条视频。每触发一条新的有效后续，推荐流总量增加 1。

示例：

| 阶段 | 推荐流内容 | 数量 |
| --- | --- | ---: |
| 初始 | A、B、D | 3 |
| 从 A 购买 X，打赏给 B | A、B、C、D | 4 |
| C 解锁广告 E | A、B、C、D、E | 5 |
| 把 E 中的 Y 打赏给 D，生成 F | A、B、C、D、E、F | 6 |

普通触发不会自动删除旧视频，推荐流会随着探索逐渐膨胀。只有达成阶段性解决时，已经过时的问题状态视频才会退出当前推荐流。

### 6.5 阶段结算与视频退场

一关由多个阶段性问题组成。玩家解决当前阶段的核心问题后，系统生成并立即播放一条阶段结果视频。

阶段结果视频播放完成后：

- 关卡配置中标记为“本阶段前置”的问题视频、已用尽商品来源和中间结果从当前推荐流消失；
- 这些视频进入历史记录，不被永久删除；
- 前置视频已解锁的商品、人物和世界状态继续保留；
- 前置视频暂时不能继续打赏；
- 阶段结果视频留在推荐流，成为新状态；
- 阶段结果视频结尾或随后出现的新内容暴露下一阶段问题。

推荐流因此形成“增长—阶段解决—收束—再次增长”的节奏，而不是只增不减。

示例：

`3 条初始内容 → 探索增长到 6 条 → 阶段解决后收束到 3 条 → 新问题出现后再次增长`

玩家回退阶段时，阶段结果视频退出，相关前置问题视频恢复到推荐流和可互动状态。

## 七、因果链与分支

### 7.1 因果关系

每条新视频都必须能回答三个问题：

1. 它由哪条视频触发？
2. 玩家送入了什么物品？
3. 它改变了谁的状态或解锁了什么内容？

如果一条视频无法清楚回答这三个问题，就不应作为物品投放的直接结果生成。

### 7.2 主线增长

主线包含关卡目标相关的所有问题、正确反馈、阶段结果和角色冲突。婚礼故障、伴娘嘲笑、新娘反击都属于主线，不因为内容偏爽文或搞笑而降为支线。

每个主线节点必须配置：

- 正确目标视频；
- 唯一正确物品；
- 正确配对生成的主线结果；
- 错误配对生成的荒诞结果；
- 是否完成当前阶段。

主线视频负责：

- 推进核心谜题；
- 解锁关键商品；
- 改变主要人物状态；
- 生成关卡收束所需的内容。

### 7.3 荒诞分支

荒诞分支只由错误的“物品 + 视频”配对触发，负责：

- 回应错误投放；
- 制造搞笑结果；
- 让玩家感觉自己的尝试被世界认真回应；
- 展示主角死亡或离开当前目标的另一条人生线。

荒诞分支不推进当前阶段，也不提供替代解法。首个原型中的荒诞分支是单条终局视频，不继续生长。

### 7.4 组合反应

同一人物先后收到不同礼物时，可以触发组合视频。

例如：

- 单独送梯子，人物爬上屋顶；
- 单独送扩音器，人物在楼下喊话；
- 已经送过梯子后再送扩音器，生成“站在屋顶公开喊话”的特殊视频。

组合反应是高价值隐藏内容，适合奖励主动试验的玩家，不要求每一对物品都有组合结果。

### 7.5 生长节点与结果节点

为了控制单关制作量，新生成的视频分为两类：

- **生长节点**：仍可接收部分礼物、解锁商品或继续生成主线；
- **结果节点**：负责展示阶段反馈或荒诞终局，播放后可以收藏、隐藏和回看，但不再接受礼物。

“每次有效打赏都生成一条新视频”不等于“每条新视频都必须无限向下分叉”。关卡设计时应明确标记节点类型，只有具有新决策价值的视频才继续生长。

## 八、推荐流超过 15 条后的整理

### 8.1 整理功能解锁

当当前关卡累计生成第 16 条视频时，解锁“不感兴趣”功能，并进行一次轻量提示：

> 推荐内容有点多？可以隐藏暂时不需要的视频。

15 条不是推荐流的硬上限。玩家可以继续保留更多内容，也可以主动整理。

### 8.2 不感兴趣

玩家点击视频菜单中的“不感兴趣”后：

- 该视频从当前推荐流中隐藏；
- 推荐算法降低同类内容的短期权重；
- 视频已经造成的世界变化不会撤销；
- 由它解锁的商品、人物和后续视频不会消失；
- 该视频不会被永久删除。

这里的“删除”是信息流层面的隐藏，不是因果层面的抹除。否则玩家可能因为删除关键来源视频而破坏关卡逻辑。

### 8.3 已隐藏内容

玩家可以从“浏览记录 / 不感兴趣”列表找回被隐藏的视频。

恢复后：

- 视频重新参与推荐；
- 原有观看记录、礼物记录和因果关系全部保留；
- 不重复触发首次观看奖励。

### 8.4 收藏

为配合信息流整理，玩家从游戏开始就可以收藏视频。收藏不改变因果状态，只帮助玩家保留重要线索。

当玩家准备隐藏一条仍有未触发关键链接的视频时，系统只提示“该视频还有未查看内容”，但允许玩家继续隐藏。因为视频可以恢复，系统不需要禁止玩家操作。

## 九、防卡死规则

为了让信息流持续增长而不破坏解谜，系统必须满足以下规则：

1. 隐藏视频不会撤销世界状态；
2. 已解锁商品不会因来源视频隐藏而重新上锁；
3. 隐藏视频可以随时恢复；
4. 重复投放不会重复扣除物品；
5. 关键教学商品保证可购买；
6. 每关提供关卡回退或完整重置；
7. 当前时间线出现互斥状态时，以最近确认的关键选择为准，并提示玩家需要回退才能体验另一结果；
8. 关卡目标所需的唯一视频被隐藏时，目标提示可引导玩家查看“不感兴趣”列表，但不自动恢复。
9. 阶段结算退场的视频可以回看，但只有回退阶段后才能重新接受礼物。

## 十、完整教学示例

### 10.1 初始推荐流

- **A：带货直播间**——主播正在演示 X，直播间有 X 的购买链接；
- **B：主人公的失败视频**——以死亡或目标彻底失败收尾，画面和评论暗示改命所需条件；
- **D：第二商品来源或主线线索**——提供另一项可能的正确条件。

玩家开局没有任何礼物。

### 10.2 标准路线

1. 玩家刷到 A，点击直播间挂链；
2. 购买 X，X 进入礼物背包；
3. 玩家回到推荐流，刷到 B；
4. 打开改命礼物栏，把 X 送回 B 的失败时刻；
5. B 暂停并播放通用礼物特效，原视频素材不改变；
6. 系统生成并立即播放预制视频 C；
7. C 展示 X 真实进入人物世界后的反应与结果；
8. 普通触发时 B 与 C 同时保留；
9. 推荐流由 A、B、D 三条增长为 A、B、C、D 四条；
10. C 中出现新线索或新广告，推动下一轮购买与投放。

### 10.3 错误路线

如果玩家把并非正确条件的 Y 送给可改命失败视频 B：

- B 产生一条完整的荒诞终局 E1；
- 推荐流由 3 条增长到 4 条；
- E1 以主人公死亡或进入另一条人生线收尾，不改变主线状态；
- 系统保证玩家仍能通过再次购买或阶段回退取得完成主线所需的正确物品 X；
- E1 进入命运记录，播放结束后返回仍可改命的 B，不能代替主线 C。

错误路线不能让玩家永久失去完成主谜题的机会。

## 十一、关卡设计数据要求

每设计一条可互动视频，需要同步填写：

- 视频 ID 与标题；
- 发布者与发生时间；
- 视频的表层笑点；
- 视频提供的线索；
- 挂载商品；
- 可接收物品清单；
- 每个物品对应的预制结果视频；
- 生成的后续视频 ID；
- 对人物状态的修改；
- 对商品和广告的解锁；
- 节点属于主线还是荒诞分支；
- 是否允许继续生长；
- 被隐藏后是否需要特殊提示。

每设计一件物品，需要同步填写：

- 物品 ID 与名称；
- 来源视频或商品链接；
- 价格与可购买次数；
- 标准用途；
- 正确的主线目标视频；
- 错误配对对应的荒诞结果；
- 特殊组合反应；
- 重复投放规则；
- 重置后的恢复规则。

## 十二、本版明确规则

- 每关可以从 3 条视频和 0 件礼物开始；
- 第一件礼物来自其中一条视频或直播的商品链接；
- 玩家购买礼物后，自行判断应投进哪条可改命失败视频；
- 无论目标是普通视频还是直播，打赏后都立即播放一条预制结果视频；
- 原视频只播放通用礼物特效，不要求即时生成演员反应；
- 正确改命生成新的主线失败状态时，B 与 C 暂时同时保留；
- 阶段结算后，已经过时的问题状态视频退出推荐流并进入历史记录；
- 视频以因果节点方式增长，并在阶段结算时收束；
- 新视频首次生成时立即播放；
- 推荐流只追踪一条当前主线，错误产生的死亡或其他人生线单独进入命运记录；
- 同一触发组合只生成一次，不刷重复视频；
- 新视频分为可继续互动的生长节点与不可继续打赏的结果节点；
- 首个原型中的荒诞分支是死亡或其他人生线的单条终局视频；
- 超过 15 条视频后解锁“不感兴趣”；
- “不感兴趣”只隐藏视频，不删除因果结果；
- 隐藏视频可以从记录中恢复；
- 已解锁商品不会因视频隐藏而丢失；
- 错误投放也应产生荒诞内容，但不能推进主线或造成永久卡关；
- 只有已经无法继续的失败视频可以接受改命礼物；
- 错误结果进入命运记录，观看后返回原失败视频继续尝试。

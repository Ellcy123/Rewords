# 《刷到你了》树状推荐流与语义物品设计规则 V0.1

> 文档性质：玩法与关卡设计规则
>
> 当前范围：初始推荐流、视频树生长、植入链接、跨视频投放、语言梗与物品回收
>
> 核心原则：**弱相关挂链，强因果回收；获得方式可以离谱，物品用途必须成立。**

### 与旧规则的关系

本文是对早期线性推荐流方案的更新。涉及以下内容时，以本文为准：

- 可接受物品的视频范围；
- 初始三条视频的内容类型；
- 资源树正确配对；
- 后续关键商品的解锁来源；
- 商品来源与剧情视频的合并；
- 物品在不同内容维度中的语义转换。

《01_推荐流生成与视频增殖规则_V0.1》中“只有可改命失败视频能接受物品”的限制不再适用。《02_第一关_婚礼关卡骨架_V0.1》中的婚礼正确路径继续保留，但初始三条视频的包装方式，以及录音笔、投影服务直接随婚礼阶段出现的方式不再适用。

---

## 一、设计目标

推荐流不应像一张按顺序发放答案的任务清单。玩家开局刷到的三条视频来自完全不同的内容维度，看起来互不相关；玩家把一条视频中取得的物品投进另一条视频后，才让内容继续生长，并逐渐发现它们能够互相解决问题。

玩家的核心乐趣来自三次意外：

1. **这个视频居然能挂这种链接；**
2. **这个东西居然能用在另一种视频里；**
3. **这个离谱用法居然真的推进了解谜。**

设计结果必须达到“第一次想不到，看到结果后立刻觉得说得通”。

---

## 二、三条初始视频是三个内容维度

第一关开局只有三条视频，但三条内容不应都是婚礼剧情、商品广告或工具教学，而应分别代表不同的短视频频道。

当前推荐结构为：

1. **都市叙事维度**：婚礼、情感、爽文或社会事件，承担关卡的表层目标；
2. **古装短剧维度**：宫斗、刺客、王爷、冷宫等竖屏短剧，适合出现时代错位的植入广告；
3. **知识博主维度**：科普、测评、调查或冷知识，适合通过术语、双关和互联网黑话改变物品含义。

三条初始视频的差异必须让玩家产生真实的“刷到下一条完全不知道会是什么”的感觉。

初始内容只是三棵树的根。第一关后续出现的关键视频、人物、服务和物品，都必须能够沿生成关系追溯回这三个根节点之一，不能在主线遇到问题时由系统凭空推送答案。

---

## 三、一条视频可以同时是剧情、结果和商品来源

游戏不单独区分“剧情视频”和“商品广告”。一条新生成的视频可以同时承担以下功能：

- 展示上一次物品投放的结果；
- 推进主线、资源线或荒诞分支；
- 出现、使用或提到一件新物品；
- 挂载该物品、人物或服务的购买链接。

因此，录音笔不必拥有一条独立且生硬的带货广告。只要某条衍生视频里出现了录音笔，或者角色说出“早知道带支录音笔”，这条视频就可以直接挂录音笔链接。

### 3.1 允许挂链的最低条件

满足以下任意一项即可挂链：

- 物品实际出现在画面中；
- 角色使用了该物品；
- 台词、字幕、评论或弹幕提到了该物品；
- 视频以赞助、同款、测评、避坑或热搜关联的形式插入该物品；
- 物品与视频中的某个词构成谐音、多义词或互联网黑话。

挂链与视频主题的相关性可以很弱，甚至应当保留明显的硬广感。弱相关负责制造荒诞，后续真实用途负责让玩家相信这不是随机乱塞。

### 3.2 植入形式

一条视频只需要一种清晰的植入方式：

- 角色口播；
- 画面中的同款商品；
- 左下角商品卡；
- 置顶评论；
- 直播间购物车；
- 突然插入的赞助字幕；
- 与关键词匹配的平台自动广告。

第一关原型中，一条生长视频最多解锁一件新的关键物品，避免玩家同时收到多条答案。

---

## 四、物品不是固定功能，而是可被语境重新解释的“词”

物品进入另一条视频后，不只按照现实物理功能结算，也可以按照名称、读音、黑话或隐喻结算。

允许使用的语义转换包括：

- **一词多义**：同一个词在不同领域表示不同事物；
- **谐音梗**：读音相近，但含义发生变化；
- **互联网黑话**：把网络术语放回字面场景，或把实体物品解释成网络工具；
- **字面化**：把抽象表达当成真实动作执行；
- **反向字面化**：把实体物品转化成抽象能力；
- **职业或服务歧义**：同一个人物在不同视频中被理解成不同工种；
- **时代错位**：现代物品进入古装戏，或古代道具进入现代内容。

### 4.1 核心示例：梯子与 VPN

古装短剧中可以出现一把实体“梯子”，并挂载“同款梯子”链接。玩家取得后：

- 把梯子投进婚礼事故视频，它按照实体含义结算，帮助伴郎够到高处的灯架；
- 把梯子投进知识博主视频，它按照互联网黑话结算，成为 VPN，让博主完成网络意义上的“翻墙”。

知识博主翻墙后生成一条新视频。他可能在海外网站刷到录音设备、引用“留证要留原声”，或直接说出录音笔，于是这条视频可以挂载录音笔链接。

这条资源链为：

`古装短剧挂出梯子 → 梯子进入知识视频后变成 VPN → 博主翻墙 → 新视频提到录音笔 → 玩家取得录音笔`

梯子的外观和名称没有被系统随意替换，变化的是它在不同内容维度中的解释方式。

---

## 五、弱相关不等于随机

每次语义转换都必须存在一座能够被玩家复述的“语言桥”。

合格结果应当能用一句话解释：

> “因为网络黑话里 VPN 叫梯子，所以知识博主拿到梯子后翻墙了。”

不合格结果只能解释为：

> “系统觉得这样比较搞笑，所以发生了。”

语言桥必须在结果视频的前五秒内被画面、字幕或台词点明。玩家可以事前猜不到，但看完后不能仍然不知道为什么。

### 5.1 意外程度分级

- **一级：现实功能**——梯子帮助够到高处；
- **二级：跨场景功能**——梯子帮助知识博主翻越实体围墙；
- **三级：语义功能**——梯子被解释为 VPN，帮助博主网络翻墙。

第一关正确资源链应至少包含一次三级意外，让玩家明确理解本作不是普通的道具组合游戏。

---

## 六、所有给出的物品都必须被回收

第一关原型不设置纯粹用于占背包、误导玩家或只触发失败的废物品。

每件可购买物品至少必须拥有一个正向配对，正向用途包括：

- 推进婚礼主线；
- 解决某棵资源树的问题；
- 生成一条携带后续关键链接的视频；
- 为取得主线必需物品创造条件。

如果一件物品只能产生荒诞终局，却无法推进任何有效路径，则应删除该物品或为它补充有效用途。

正确用途也必须有梗。单纯出现“物品有效，问题解决”不够，应至少包含一次认知落差，例如：

- 物品解决了字面问题，却暴露出更深一层问题；
- 物品在另一内容维度中改变含义；
- 植入方式极其生硬，但回收方式非常准确；
- 玩家以为得到的是工具，实际得到的是人物、服务或权限。

### 6.1 一件物品可以拥有多个有效用途

同一物品允许同时服务不同树枝。例如梯子既能推进婚礼事故线，也能推进知识博主资源线。

这类“一物多用”优先于不断增加新物品，因为它能让玩家重新审视已经刷过的视频，并提高自由探索的价值。

---

## 七、推荐流的树状生长规则

玩家体感应当是三棵内容树逐渐展开，底层结构则是一张由跨视频投放连接起来的有向因果网。

### 7.1 基本生成公式

`来源视频 + 投入物品 → 一条预制结果视频`

新视频必须记录：

- 它由哪条视频生成；
- 玩家投入了什么；
- 该结果属于主线推进、资源推进还是荒诞终局；
- 视频中出现或提到了什么新物品；
- 它因此挂载了什么新链接。

### 7.2 三类结果

1. **主线生长节点**：改变婚礼命运，生成新的主线状态；
2. **资源生长节点**：不直接解决婚礼，但解锁后续必需物品；
3. **荒诞终局节点**：形成死亡或另一种人生，不再继续生长。

此前“只有婚礼视频存在正确配对”的规则在此处放宽：商品视频、古装短剧、知识视频和直播间也可以拥有自己的正确配对。它们的正确结果不推进婚礼状态，而是推进资源树。

### 7.3 关键物品不得凭空出现

后续主线所需的录音笔、投影服务等关键内容，必须由玩家主动展开某条已有树枝后解锁。系统不能因为 W300 出现造谣问题，就自动推送录音笔广告。

系统可以调整新视频的推荐权重，但不能替玩家完成解谜因果。

## 八、第一关的示意结构

以下结构只用于说明规则，不替代后续的完整节点表与视频脚本。

```text
初始三条视频
├─ W001 都市婚礼短剧：新娘被灯架砸死
├─ C001 古装短剧：出现并挂载“梯子”
└─ K001 知识博主：提供知识区语境与其他弱相关植入

C001 取得梯子
├─ 梯子 → W001
│  └─ W101 伴郎够到灯架，但不会修
│     └─ 继续寻找空调师傅
│
└─ 梯子 → K001
   └─ K101 梯子被解释为 VPN，博主网络翻墙
      └─ K102 视频中出现或提到录音笔
         └─ 解锁录音笔链接
```

玩家会先把这理解为几条互不相关的搞笑视频，之后才发现知识区取得的录音笔能够被送回婚礼线，形成跨维度回收。

## 九、单个节点的设计模板

设计每条可生长视频时，必须填写以下内容：

| 字段 | 说明 |
|---|---|
| 节点 ID | 视频的唯一编号 |
| 内容维度 | 都市叙事、古装短剧、知识博主或其他频道 |
| 当前事件 | 五秒内能够理解的核心事件 |
| 可投入物品 | 当前已解锁且允许尝试的物品 |
| 正向配对 | 能推进主线或资源树的配对 |
| 语言桥 | 谐音、多义词、黑话或现实功能的解释 |
| 结果视频 | 投放后立即播放的预制视频 |
| 新植入 | 结果中出现或提到的新物品 |
| 新链接 | 因植入而开放的购买内容 |
| 错误结果 | 其他配对对应的荒诞终局 |
| 回收位置 | 本节点提供的物品将在哪条有效路径中用上 |

“新植入”和“回收位置”不可同时为空。否则该节点既不继续生长，也没有承担有效回收，应当改为明确的终局节点。

---

## 十、验收清单

每设计一件物品或一条生长视频，都要检查：

- [ ] 三条初始视频是否属于明显不同的内容维度；
- [ ] 所有关键节点是否能够追溯回初始三条视频；
- [ ] 新商品是否通过某条实际视频出现或被提到；
- [ ] 商品链接是否允许带有故意生硬的植入感；
- [ ] 每件可购买物品是否至少有一个正向用途；
- [ ] 正向用途是否包含明确的笑点或认知落差；
- [ ] 谐音或多义词是否存在一句话可解释的语言桥；
- [ ] 语义桥是否能在结果视频前五秒内被理解；
- [ ] 是否至少有一件物品拥有两个不同维度的有效用途；
- [ ] 是否至少有一次三级语义意外；
- [ ] 关键商品是否由玩家展开树枝获得，而非系统按当前问题自动发放；
- [ ] 新视频是否仍遵守核心事件前置、总长不超过十五秒的规则。

任意一项未通过，都应先修改节点关系或笑点，再进入完整脚本设计。

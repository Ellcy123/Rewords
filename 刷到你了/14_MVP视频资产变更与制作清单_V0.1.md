# 《刷到你了》MVP 视频资产变更与制作清单 V0.1

> 文档性质：AI 私聊主循环改版后的视频资产影响与制作规格
> 制定日期：2026 年 7 月 20 日
> 对应策划：《13_AI私聊主循环_MVP完整策划案_V0.2》
> 当前状态：开发与视频生产共同执行的资产基线

## 一、变更结论

AI 私聊进入新娘改命主循环后，婚礼后半段的物品来源发生了变化，但婚礼事件本身没有推翻。因此本次不是重拍完整主线，而是：

- 复用 14 条现有视频；
- 修改 1 条现有视频 `K101`；
- 新增 4 条娱乐视频或动态分镜 `E001`、`E101`、`E102`、`E201`；
- 增加 1 个送回过去的 UI 过渡，不单独生产视频；
- 保留全部现有荒诞误投分支。

第一轮开发不等待正式视频。四个 E 节点和新版 `K101` 先使用明确标记的 CSS 动态分镜验证玩法；闭环试玩成立后，再替换为正式竖屏视频。

## 二、新旧流程对视频的影响

### 2.1 旧流程

`W001 → W101 → W300 → W301 → W400`

录音笔原本由 `K101` 的广告直接提供，投影服务由 `C101` 提供。

### 2.2 新流程

`W001 → W101 → W300`，同时解锁 `E001`；

`E001 → E101 或 E102 → AI 私聊 → E201 → 可重复购买录音笔`；

获得录音笔后，两条投放顺序可以互换：

- `C001 + 录音笔 → C101 → 购买投影服务`；
- `W300 + 录音笔 → W301`；

两条都完成后：`W301 + 投影服务 → W400`。

新流程改变的是录音笔的来源和获得顺序：

- `K101` 从商品来源改为公共线索；
- `E201` 成为录音笔唯一首次来源；
- `C101` 仍然提供投影服务，玩家必须在获得录音笔后通过 `C001 + 录音笔` 解锁；
- 玩家至少需要从 `E201` 购买两支录音笔，一支用于 `C001`，一支用于 `W300`；
- `W300`、`W301`、`W400` 的视频因果仍然成立，无需重拍。

## 四、`K101` 修改规格

### 4.1 修改目标

保留“梯子在知识区被理解成 VPN”的语言梗，但移除它作为录音笔商品来源的职责。

新版内容只负责告诉玩家：被剪辑的结论不能代替完整原始数据。这条认知会在炎鑫遭遇恶意剪辑时产生呼应，但玩家不能在这里直接买到录音笔。

### 4.2 建议文案

- 标题：`梯子变成 VPN`
- 结果字幕：`梯子连接成功`
- 解释字幕：`翻过墙，只看十秒结论还是会被骗`
- 评论字幕：`数据能打开，完整上下文还得自己留`
- 视频结尾：页面展示被截断的跑分结论与“查看原始记录”提示，不出现任何具体录音设备。

### 4.3 动态分镜节奏

| 时间 | 画面与信息 |
| --- | --- |
| 0–2 秒 | 实体梯子被拖进知识区，画面跳出“VPN 已连接” |
| 2–5 秒 | 海外跑分页面打开，只显示经过截取的十秒结论 |
| 5–7 秒 | 博主点开“查看原始记录”，发现完整记录未保存 |
| 7–9 秒 | 字幕落点：“翻过去了，不等于看完整了。” |

### 4.4 文件策略

- 旧文件 `K101_ltx_raw_v1.mp4` 和旧封面保留在素材历史中，不继续作为游戏运行资产；
- 正式新版命名为 `K101_ltx_raw_v2.mp4`、`K101_thumbnail_v2.jpg`；
- 新版素材完成前，游戏中把 `K101` 明确显示为动态分镜；
- 不允许只移除小黄车按钮，却继续播放带录音笔广告的旧视频。

## 五、新增娱乐视频规格

### 5.1 `E001`：PK 只剩最后 30 秒

**职责：** 提供上票或不上票之前的共用场景，不承担选择结果。

**建议时长：** 10–14 秒。

| 时间 | 画面与信息 |
| --- | --- |
| 0–3 秒 | 竖屏 PK 双人画面，炎鑫比分落后，UI 显示最后 30 秒 |
| 3–6 秒 | 对手提前庆祝；弹幕刷“又是剧本吧”“上次视频只放了十秒” |
| 6–10 秒 | 炎鑫嘴硬：“输就输，别乱花钱。”同时明显盯着比分 |
| 10–14 秒 | 视频停在关键时刻，UI 弹出“上票”与“先不上”判断题 |

倒计时、比分、礼物按钮和弹幕优先由 UI 层实现，视频素材只提供人物动作和场景，方便复用。

### 5.2 `E101`：上票结果

**职责：** 展示外部结果和公开感谢，不直接判定恋爱关系。

**建议时长：** 5–8 秒。

- 礼物效果扫过画面；
- 比分在最后一秒反超；
- 炎鑫先愣一下，再对镜头说：“行，赢了。某个人别装路过，我看见了。”；
- 结尾保留一瞬间担心玩家是否花钱过度的表情，为私聊中的边界压力留空间。

### 5.3 `E102`：不上票结果

**职责：** 展示落败结果，但不把不上票包装成错误答案。

**建议时长：** 5–8 秒。

- PK 倒计时归零，炎鑫落败；
- 他关掉庆祝音效，说：“输了就输了，至少这次别只剪最后十秒。”；
- 看向在线名单或熟悉的头像，表现他知道玩家在场；
- 不出现责怪、掉好感、红色失败提示。

### 5.4 `E201`：这次我从开播前录到下播后

**职责：** 展示炎鑫完成取证任务的实际行动，并自然产生录音笔小黄车。

**建议时长：** 15–20 秒。

| 时间 | 画面与信息 |
| --- | --- |
| 0–3 秒 | 炎鑫把带摄像头的录音笔放到桌上：“这次提前开。” |
| 3–7 秒 | 设备同时记录开播前后台声音、直播画面和时间信息 |
| 7–11 秒 | 对比恶意十秒剪辑与完整记录，前后语义明显相反 |
| 11–15 秒 | 炎鑫引用玩家建议：“有人说，别只保存你想证明的那十秒。” |
| 15–20 秒 | 他完成公开回应，画面出现同款录音笔小黄车 |

视频只做一个公共版本。上票和不上票的关系差异由此前私聊、人物称呼、可替换字幕和最终来信承担，不为两条路线各生产一条完整 `E201`。

## 六、送回过去的 UI 过渡

玩家是在炎鑫发布 `E201` 后取得录音笔，但要把它用于更早发生的 `W300` 握手现场。需要在投放时解释时间关系，避免玩家误以为设备在谣言发生后才开始记录。

投放录音笔后、播放 `W301` 前，增加 1.5–2 秒通用改命过渡：

> `把录音笔送回握手发生之前`

随后表现时间轴回卷到新娘与维修人员握手前。该过渡使用现有 UI 和粒子效果，不单独制作真人视频。

## 七、最低制作量与优先级

### P0：可玩性验证

- `K101` 动态分镜新版；
- `E001` 动态分镜；
- `E101` 动态分镜；
- `E102` 动态分镜；
- `E201` 动态分镜；
- 录音笔送回过去的 UI 过渡。

完成 P0 即可验证完整玩法，不等待新增真人或 AI 生成视频。

### P1：正式视频替换

- 制作 `K101_v2`；
- 制作一条 PK 共用前段；
- 制作两条短结果结尾；
- 制作一条录音笔测试视频；
- 为四个 E 节点制作封面。

### P2：体验增强

- 根据玩家称呼替换 `E201` 的 UI 字幕；
- 增加评论区对完整视频与恶意剪辑的讨论；
- 增加炎鑫在公开感谢和私聊之间的态度反差；
- 根据试玩结果决定是否制作跳舞、全场感谢和安可节点。

## 九、资产验收标准

- `K101` 的画面、字幕、评论和按钮中都不存在录音笔广告或购买入口；
- 录音笔只能在 `E201` 首次发现和购买；
- `E001` 的上票与不上票共用前半段，选择后进入明显不同的短结果；
- `E102` 不使用失败色、错误音效或责怪文案；
- `E201` 在小黄车出现前已经完整展示设备与人物任务的关系；
- `W300 → W301` 之间能够理解“把设备送回握手前”的时间逻辑；
- 六条现有荒诞误投视频继续可播放，其中 `X016` 允许玩家在获得录音笔后通过额外购买探索；
- 只有当前必需的两支录音笔和投影服务可以触发结算补助，探索 `X016` 所需的额外录音笔不能用补助购买；
- AI 服务关闭时，所有视频节点仍可按相同顺序解锁；
- 正式素材未完成时只显示标记清楚的动态分镜，不回退播放逻辑冲突的旧素材。

# 《刷到你了》手机局域网测试手册

**启用日期：** 2026-07-20  
**默认端口：** 4173

## 开发测试

在当前工作副本的 `刷到你了/web-prototype` 中执行：

```powershell
npm run dev -- --host 0.0.0.0 --port 4173
```

手机与电脑连接同一个普通 Wi-Fi，然后打开终端本次显示的 `Network` 地址，例如 `http://192.168.1.23:4173/`。电脑重连网络后 IP 可能变化，必须重新查看，不沿用旧地址。

## 阶段验收

```powershell
npm run typecheck
npm test -- --run
npm run build
npm run preview -- --host 0.0.0.0 --port 4173
```

若 4173 被占用，依次尝试 4174、4175，并在记录中填写实际端口。需要全新存档时使用无痕窗口，或清除这个地址的网站数据。

## 同网打不开的排查顺序

1. 确认命令包含 `--host 0.0.0.0`，服务没有只监听 `localhost`。
2. 用 `ipconfig` 重新确认当前 Wi-Fi IPv4，并使用终端实际 Network 地址。
3. 确认手机与电脑连接同一普通 Wi-Fi，不是访客网络；关闭手机流量、VPN 和代理后重试。
4. 确认 Windows 当前网络为“专用网络”，并允许 Node.js/对应端口通过专用网络防火墙。
5. 检查端口是否被其他进程占用；必要时改用 4174 或 4175。
6. 电脑本机先访问同一个局域网地址；本机也打不开时优先检查服务和端口，只有手机打不开时优先检查网络隔离与防火墙。

## 测试记录模板

```markdown
### 手机测试记录

- 日期：
- 测试提交：
- 手机 / 系统 / 浏览器：
- 网络类型：同一 Wi-Fi / 其他
- 实际地址：192.168.1.x:4173
- 全新存档：是 / 否
- 测试路线与关键选择：
- 结果：PASS / FAIL
- 问题与复现步骤：
- 截图或录屏路径：
```

局域网 IP 可以在对外记录中写成 `192.168.1.x:4173`。公网隧道不是默认测试方式，只有手机与电脑确实不能处于同一网络时再考虑。

# 《刷到你了》第一关：婚礼关卡骨架 V0.1

> 文档性质：首个原型关卡设计  
> 对应规则：《01_推荐流生成与视频增殖规则_V0.1.md》  
> 当前制作范围：只制作这一关，不展开账号经营、乙游系统或第二关  
> 当前细化范围：关卡目标、玩家视角、内容生产规则、第一阶段空调谜题、后续爽文方向

## 一、关卡定位

第一关是一场从小麻烦逐渐升级的荒诞婚礼。

前期必须看起来简单、日常，甚至有点无聊：婚礼现场空调坏了。玩家很快找到梯子和空调师傅，看似轻松解决问题。阶段结果出现后，旧问题视频消失，但新的麻烦立即暴露，婚礼不断以“终于好了——等等，又出事了”的节奏向前推进。

关卡体验由三部分组成：

- **解谜**：在推荐流中寻找商品或服务，把它投给正确的视频；
- **爽文**：玩家以新娘身份面对嘲笑和欺负，选择不同方式反击；
- **荒诞反馈**：商城可以买到的不只是物品，还可以是空调师傅等人物或服务。

## 二、玩家身份与目标

### 2.1 玩家身份

在婚礼关中，玩家代入新娘。

玩家不是控制新娘行走或直接选择台词，而是通过推荐流为新娘取得物品、人物和服务。结果视频负责展示新娘如何使用玩家送入的内容。

因此玩家的每次打赏同时具有两层含义：

- 玩法层：向某个视频投入解谜要素；
- 角色层：为“自己”取得解决问题或反击他人的手段。

### 2.2 明示目标

关卡开始时只显示一句结果要求：

> **让婚礼顺利结束。**

不显示目标视频封面，不公布需要经历多少阶段，也不直接告诉玩家哪件物品是正确答案。

### 2.3 情绪目标

除了明示目标，关卡还有一个不写在任务栏里的情绪目标：

> **今天谁也别想让我受气。**

玩家可以选择直接、体面或荒诞的反击方式。这些差异产生不同结果视频和支线反馈，但首版不制作完整的长期人设养成系统。

## 三、视频与直播的统一生产规则

第一关的所有剧情反应都使用预先制作好的视频资产。

### 3.1 普通视频

玩家向普通视频打赏后：

1. 原视频暂停；
2. 界面播放通用礼物特效；
3. 原视频素材本身不改变；
4. 特效结束后立即播放一条新的预制结果视频；
5. 普通触发时原视频与结果视频同时保留；
6. 阶段结算时，已过时的前置视频退出推荐流。

### 3.2 直播间

直播间由预制视频与直播 UI 包装组成，不要求演员对玩家礼物进行实时表演。

玩家向直播间打赏后：

1. 直播素材暂停并播放通用礼物特效；
2. 特效结束后立即播放一条新的预制视频；
3. 新视频可以包装成直播切片、下播回应或主播刚发布的内容；
4. 直播间本身可以继续保留，除非阶段结算将其归档。

无论目标是普通视频还是直播间，每次有效打赏都统一产出 **1 条预制结果视频**。

## 四、可购买内容

第一关商城中的“礼物”分为三类：

- **物品**：梯子、香槟、礼花炮、录音笔等；
- **人物**：空调师傅、司仪、保安等；
- **服务**：紧急维修、现场投影、同城配送等。

人物和服务也以商品链接形式购买，并像礼物一样投入视频。荒诞感来自平台把一个完整的人或一项服务包装成可打赏商品。

购买人物不表示玩家拥有这个人，只表示获得一次“让该人物进入目标视频并提供服务”的投放机会。

## 五、初始推荐流

关卡初始只有 3 条内容。

### W001：婚礼怎么这么热

**类型**：普通视频  
**账号**：婚礼跟拍账号  
**作用**：核心目标视频、第一阶段问题  

视频内容：

- 婚礼即将开始，场地空调坏了；
- 新娘妆容开始融化，宾客拿菜单扇风；
- 一名刻薄宾客或伴娘嘲笑新娘选了廉价场地；
- 画面能看到空调位置很高；
- 评论区有人说“这么高，先找个梯子吧”。

### A001：这梯子什么都能够到

**类型**：商品视频或工具直播  
**挂链**：伸缩梯  
**作用**：提供第一件物品  

伸缩梯为免费教学商品或使用初始赠币购买，保证玩家开局即可取得。

### L001：同城万能维修直播间

**类型**：预制直播间  
**挂链**：空调师傅上门一次  
**作用**：提供第一位可购买人物  

直播内容以一本正经的方式售卖“空调师傅本人”，强化游戏的荒诞规则。购买后，背包中出现一次性礼物“空调师傅”。

## 六、第一阶段核心谜题

### 6.1 阶段问题

空调位置太高，同时现场没有会维修的人。玩家需要让“梯子”和“空调师傅”都进入婚礼事件。

这是一道双要素组合谜题：

`伸缩梯 + 空调师傅 + 婚礼视频 = 空调修好`

两件内容可以用不同顺序送入，路线最终汇合到同一阶段结果。

### 6.2 路线一：先送梯子

1. 玩家购买伸缩梯；
2. 把梯子打赏进 W001；
3. 立即播放 W101《伴郎上去了，但他不会修》；
4. 视频中伴郎爬上梯子，摆弄空调后被卡在吊顶边；
5. W001 与 W101 暂时同时保留；
6. 玩家购买空调师傅；
7. 把空调师傅送进 W101；
8. 立即播放阶段结果 W200《师傅到了，空调终于好了》。

### 6.3 路线二：先送空调师傅

1. 玩家购买空调师傅；
2. 把空调师傅打赏进 W001；
3. 立即播放 W102《师傅来了，但他够不着》；
4. 视频中师傅站在过高的空调下，认真等待梯子；
5. W001 与 W102 暂时同时保留；
6. 玩家购买伸缩梯；
7. 把梯子送进 W102；
8. 立即播放阶段结果 W200《师傅到了，空调终于好了》。

### 6.4 汇合规则

W101 和 W102 是两条不同的生长节点，W200 是共同的阶段结果节点。

这让玩家感觉顺序自由，同时把制作量控制在合理范围。关卡不需要为所有操作顺序制作完全独立的后半段，两条路线在阶段解决时汇合。

## 七、阶段结算

W200 播放完成后，第一阶段正式结算。

推荐流发生以下变化：

- W001 从当前推荐流退场；
- 已经出现的 W101 或 W102 从推荐流退场；
- A001 与 L001 作为已经完成教学职责的商品来源，从推荐流退场；
- 退场视频进入历史记录，可以回看但不能继续打赏；
- W200 留在推荐流，成为婚礼的当前状态；
- 梯子和空调师傅保持已解锁状态；
- 玩家回退第一阶段时，W200 退出，相关前置视频恢复。

阶段结算后的推荐流数量会下降，形成清晰的“问题已解决”反馈。

## 八、前期爽文方向

空调问题只是简单教学。关卡后续会把矛盾从“解决现场故障”转向“新娘如何面对欺负”。

本版采用一名刻薄伴娘作为前期对手。她继续嘲笑新娘寒酸，认为连空调师傅都是靠网络打赏才请得起。

推荐流随后可以出现以下商品：

- **香槟**：让新娘直接泼回去；
- **礼花炮**：制造无伤害的夸张爆炸效果，把对方炸成爆炸头；
- **蛋糕喷射器**：把冲突变成抽象网络名场面；
- **录音笔或投影服务**：公开对方之前说过的话，完成体面反杀。

每种反击都生成不同的预制结果视频。首版不制作跨关卡人格系统，但结果可以在本关内影响评论区、推荐广告、宾客态度和最终婚礼视频。

## 九、分支控制

第一关追求高自由度，但所有分支都必须人工设计和制作视频，因此采用以下控制规则：

- 每件关键内容优先设计 1 个主线用法、1 个有效支线用法和 1 个纯玩梗用法；
- 生长节点可以继续接收部分礼物；
- 结果节点只负责播放、收藏、隐藏和回看，不再接受礼物；
- 普通玩梗支线最多连续生长 2～3 层；
- 不同操作顺序尽量在阶段结果处汇合；
- 阶段结算会让已过时视频退场，防止推荐流无限膨胀；
- 非标准投放也必须产生视频反馈，但不保证每条反馈继续分叉；
- 关键物品误投后，玩家仍能通过再次购买、支线奖励或阶段回退取得主线所需内容。

## 十、首关制作预算

当前项目只制作和验证婚礼关。

首版建议把总量控制在以下范围：

- 初始内容：3 条；
- 主线生长与阶段结果：8～12 条；
- 爽文反击路线：6～10 条；
- 误投和纯玩梗反馈：6～10 条；
- 广告、商品演示和直播包装：4～6 条；
- 全关独立预制视频总量：约 27～35 条。

玩家单次通关预计看到 15～22 条内容，其余用于替代路线、误投和重玩发现。

该预算是内容上限，不是必须填满的指标。原型首先验证 3 条初始内容、2 件关键礼物、2 种操作顺序和 1 次阶段结算是否有趣。

## 十一、本版已经锁定的规则

- 当前只设计和制作婚礼关；
- 玩家在这一关代入新娘；
- 关卡明示目标只有“让婚礼顺利结束”；
- 开局推荐流只有 3 条内容；
- 开局可以很快取得梯子；
- 另一条直播提供可购买的空调师傅；
- 人物与服务可以像物品一样被购买和打赏；
- 直播间与普通视频都使用预制素材；
- 每次有效打赏都立即自动播放 1 条预制结果视频；
- 原视频不即时生成演员反应；
- 普通触发让前后视频同时存在；
- 阶段解决后，已经过时的前置视频退出推荐流；
- 退场视频进入历史记录，回退阶段后才能重新互动；
- 梯子和空调师傅可以用不同顺序投入，并汇合到同一个修复结果；
- 后续会加入被嘲笑后的爽文反击分支；
- 结果视频分为可继续生长与不可继续打赏两类。

## 十二、本版刻意不展开的内容

为了先验证核心自由度，本版不设计以下系统：

- 第二关及后续章节；
- 完整账号模拟经营；
- 完整乙游和男模好感系统；
- 跨关卡人格标签；
- 婚礼关后半段的全部事故链；
- 最终结局数量与完整视频树；
- 商业化、真实付费与联网功能。

后续讨论将继续围绕婚礼关逐阶段补全，不扩大到新的系统或关卡。

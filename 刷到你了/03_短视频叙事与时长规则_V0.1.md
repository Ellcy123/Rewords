# 《刷到你了》短视频叙事与时长规则 V0.1

> **脚本更新提示**：本文继续作为全项目的时长与节奏规范；第一关各节点的最新秒数和逐镜剧情，以 `05_第一关_婚礼逆天改命/剧情节点/` 中的独立文件为准。本文后半部分的第一关示例不再作为节点脚本事实来源。

> 文档性质：视频内容制作规范  
> 适用范围：第一关全部普通视频、直播素材、商品视频、主线结果和荒诞终局  
> 核心原则：核心事件前置，5 秒讲清，15 秒封顶

## 一、目标

游戏必须保留刷短视频的速度感。玩家不应该为了理解一个节点，被迫观看传统影视式的铺垫、对话和慢节奏表演。

每条视频都必须做到：

1. 第一眼就有事发生；
2. 5 秒内理解这条视频的核心结果；
3. 15 秒内结束；
4. 剩余信息可以在结果之后回头补充；
5. 玩家静音观看时仍能理解主要内容。

## 二、硬性时长规则

### 2.1 五秒核心

视频播放到第 5 秒时，玩家必须已经能够回答：

- 谁出事了；
- 发生了什么；
- 结果为什么无法继续，或本次改命成功了什么。

如果第 5 秒仍然只是在展示场景、寒暄、人物入场或气氛铺垫，该视频不合格。

### 2.2 十五秒封顶

首个原型中的每条独立视频资产不得超过 15 秒。

推荐时长：

- 普通主线视频：8～13 秒；
- 商品视频：6～10 秒；
- 直播素材：8～12 秒；
- 荒诞终局：6～12 秒；
- 阶段结果：8～15 秒。

15 秒是上限，不是目标。一个笑点 7 秒能讲完，就不拉长到 15 秒。

### 2.3 一个视频只讲一个核心事件

一条视频只承担一个主要结果，例如：

- 新娘被灯架砸死；
- 伴郎拧错卡扣，新娘仍然死亡；
- 师傅固定灯架，新娘存活；
- 伴娘造谣，婚礼取消；
- 录音到手，但现场没人相信；
- 证据被投上大屏，婚礼继续。

不能在同一条视频里同时完成两个主线投放步骤。

## 三、核心事件前置

### 3.1 先给结果，再解释原因

视频不要求按照现实时间顺序剪辑。优先在开头展示最强结果，然后使用回放、字幕、评论或快速倒放补充原因。

推荐结构：

| 时间 | 内容职责 |
| --- | --- |
| 0～1 秒 | 最强画面、结果或冲突标题 |
| 1～3 秒 | 明确人物与问题 |
| 3～5 秒 | 确认结果，以及为什么剧情无法继续 |
| 5～10 秒 | 快速回放原因、补充笑点或展示物品作用 |
| 10～15 秒 | 留下下一件物品、下一条视频或评论线索 |

### 3.2 禁止慢铺垫

以下内容不能占用视频开头：

- 空镜和场地全景；
- 主播说“家人们大家好”；
- 人物完整自我介绍；
- 重复说明上一条视频已经发生的事；
- 为了展示演员表情而长时间停顿；
- 与核心事件无关的台词往返。

必要背景使用一行字幕、账号标题或 1 秒闪回解决。

### 3.3 信息可以后补

玩家先看到“新娘死了”，再知道灯架为什么掉落；先看到“婚礼取消”，再知道伴娘剪辑了什么。

后补信息可以通过：

- 画面回放；
- 局部特写；
- 评论区截图；
- 新闻标题；
- 账号发布文案；
- 时间戳；
- 结尾一闪而过的商品或人物。

## 四、不同视频类型模板

### 4.1 主线失败视频

适用于死亡、婚礼取消和其他无法继续的节点。

建议结构：

1. 0～2 秒：直接展示死亡或失败结果；
2. 2～5 秒：确认主角、目标和失败状态；
3. 5～10 秒：回放关键原因；
4. 10～15 秒：给出可观察线索，不直接显示答案。

失败视频必须让玩家产生：“如果当时多了某样东西，结果可能不同”的判断。

### 4.2 正确改命的中间视频

适用于 W101、W301 等已经加入一个正确条件、但仍然失败的节点。

建议结构：

1. 0～2 秒：展示“还是失败了”；
2. 2～5 秒：展示上一件正确物品确实发挥了作用；
3. 5～10 秒：揭示还缺少的下一项能力；
4. 10～15 秒：用评论或画面把线索指向下一商品来源。

中间视频不能让玩家误以为上一件物品完全错误。它必须同时传达“方向对了”和“条件还不够”。

### 4.3 阶段成功视频

适用于 W200、W400 等阶段结果。

建议结构：

1. 0～2 秒：直接展示主角存活或婚礼恢复；
2. 2～5 秒：明确正确物品如何改变结果；
3. 5～10 秒：快速回放完整改命过程；
4. 10～15 秒：留下下一阶段的不安因素或人物动作。

### 4.4 荒诞终局

荒诞终局也必须先给结局，不能先解释玩家为什么送错。

建议结构：

1. 0～2 秒：展示新死亡方式或另一条人生线的最终状态；
2. 2～5 秒：让玩家看懂错误物品造成了什么；
3. 5～10 秒：完成主要笑点；
4. 10～12 秒：显示“婚礼线终止”或同类命运标签。

荒诞终局不留新的主线悬念，播放结束后返回原失败节点。

### 4.5 商品视频

商品视频的任务不是完整带货，而是让玩家迅速理解“这是什么、能干什么”。

建议结构：

1. 0～2 秒：商品或人物直接占据画面中心；
2. 2～5 秒：展示唯一核心能力；
3. 5～8 秒：补一个荒诞卖点或使用笑点；
4. 8～10 秒：商品链接停留，允许玩家点击。

例如空调师傅直播开头不说欢迎词，第一帧直接出现：

> “空调、吊顶、灯架，我都能上。”

### 4.6 直播素材

直播只是预制视频加 UI 包装，同样遵守 5 秒核心。

- 不播放等待观众进入的过程；
- 不展示无信息量的弹幕闲聊；
- 开头直接进入商品、事故或争执；
- 弹幕只补充线索、笑点和误导；
- 直播播放到失败终点后，才开放改命礼物栏。

## 五、字幕与静音理解

多数信息必须不依赖声音才能理解。

### 5.1 字幕规则

- 一条字幕尽量不超过 14 个汉字；
- 同时最多显示两行；
- 核心结果使用标题式短句；
- 对话字幕只保留推动事件的半句；
- 不把长篇剧情说明塞进评论区。

### 5.2 视觉规则

每条视频至少使用一种无声也能理解的表达：

- 结果标题；
- 人物关系标签；
- 箭头、圈选或局部放大；
- 时间前后对比；
- 假视频与原视频并排；
- “婚礼取消”“婚礼线继续”等状态字幕。

## 六、婚礼关关键节点时长示例

### W001《婚礼灯架事故》：12 秒

- 0～1 秒：灯架坠落，画面黑掉；
- 1～3 秒：字幕“婚礼当天，新娘被灯架砸死”；
- 3～5 秒：新闻式画面确认婚礼终止；
- 5～8 秒：快速回放灯架晃动却无人发现；
- 8～12 秒：特写高处固定件，评论飘过“这么高谁上得去”。

### W101《伴郎拧错了》：11 秒

- 0～2 秒：灯架再次坠落，字幕“有梯子，还是死了”；
- 2～5 秒：伴郎站在梯子上拧错固定件；
- 5～8 秒：特写承重卡扣被松开；
- 8～11 秒：评论提示“这得找懂吊顶的人”。

### W200《新娘活下来了》：12 秒

- 0～2 秒：新娘站在完好的灯架下继续仪式；
- 2～5 秒：师傅踩梯子锁紧承重卡扣；
- 5～8 秒：快速对比原死亡画面与当前结果；
- 8～12 秒：伴娘躲在角落偷拍视频。

### W300《婚礼当天私会维修工？》：13 秒

- 0～2 秒：新娘离场，字幕“婚礼取消”；
- 2～5 秒：大屏播放被剪成暧昧效果的视频；
- 5～9 秒：回放新娘只是感谢师傅；
- 9～13 秒：伴娘带节奏，宾客开始指责新娘。

### W301《有录音也没人信》：11 秒

- 0～2 秒：新娘举着录音笔，婚礼仍然取消；
- 2～5 秒：录音播放伴娘承认恶意剪辑；
- 5～8 秒：现场太吵，宾客听不清；
- 8～11 秒：画面扫到婚礼大屏，暗示需要投影服务。

### W400《全场看见真相》：12 秒

- 0～2 秒：大屏并排播放原视频与假视频；
- 2～5 秒：伴娘的密谋录音同步响起；
- 5～8 秒：伴娘被揭穿并离场；
- 8～12 秒：新娘重新走向舞台，字幕“婚礼线继续”。

## 七、脚本验收清单

每条视频进入制作前必须逐项通过：

- [ ] 第一秒有结果、冲突或强视觉信息；
- [ ] 第五秒前能说清核心事件；
- [ ] 总时长不超过 15 秒；
- [ ] 一条视频只承担一个核心结果；
- [ ] 静音仍能理解主要信息；
- [ ] 没有无信息量的开场和寒暄；
- [ ] 主线失败视频明确说明为什么无法继续；
- [ ] 中间失败视频同时体现“前一步正确”和“条件仍不足”；
- [ ] 商品视频 5 秒内讲清唯一能力；
- [ ] 结尾线索不直接公布下一步答案；
- [ ] 结果视频与对应节点规则一致。

任何一项未通过，都先修改脚本，不进入拍摄或视频生成。

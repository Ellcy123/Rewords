# W101 RunningHub 图生视频试片设计

> 日期：2026-07-18
>
> 状态：方案已确认
>
> 目标：沿用 W001 的婚礼视觉连续性，制作一条表现“有梯子，但不会修”的 W101 无字幕视频，并用独立 JSON 时间轴配置字幕

## 1. 试片范围

本次只制作 `W101《有梯子，但不会修》` 的一条 9:16 竖屏试片，不修改剧情树或前端播放器。

试片采用“基于 W001 关键帧编辑 → RunningHub LTX2.3 图生视频 → 独立字幕配置”的方案。默认只提交一次付费视频任务；仅当结果缺少梯子、缺少伴郎维修动作、严重变形或完全偏题时才允许第二次任务，总计不超过两次。

## 2. 叙事目标

画面必须同时表达：

1. 梯子已经解决了“够不到灯架”的问题；
2. 伴郎缺少专业能力，拧错零件导致安全卡扣弹开；
3. 灯架再次失稳，新娘仍然面临死亡结果。

视频主体约 8 秒：

```text
0～2 秒：伴郎站在梯子顶端，自信地拧动错误零件
2～5 秒：真正的安全卡扣弹开，灯架开始倾斜
5～7 秒：伴郎慌张回头，新娘和宾客迅速躲避
7～8 秒：灯架继续下坠，以白闪结束
```

## 3. 关键帧设计

以 `刷到你了/assets/video-tests/W001/W001_keyframe_v1.png` 作为视觉参考和编辑目标，保留婚礼场地、暖色灯光、黑色灯架、新娘造型与拟真人短剧质感。

关键帧新增：

- 一把清楚、完整、结构可信的银色伸缩梯；
- 一名穿浅灰衬衫和深色西裤的年轻伴郎站在梯子上；
- 伴郎一手稳定身体，一手伸向灯架上的错误旋钮或卡扣；
- 灯架略微失稳，但尚未发生直接撞击；
- 新娘位于下方或侧下方抬头观察，宾客开始察觉异常。

画面中梯子、伴郎手部、错误零件和灯架必须在空间上可读。不得出现文字、平台 UI、品牌、水印、血液、伤口或直接撞击。

## 4. 视频生成设计

继续使用已验证的 RunningHub AI 应用：

- WebApp ID：`2031277395491164161`；
- 图片节点：`3/image`；
- 提示词节点：`65/text`；
- 时长节点：`83/value=8`；
- 模式节点：`219/value=false`。

视频运动重点：伴郎错误转动零件；相邻安全卡扣突然弹开；灯架加速倾斜；伴郎抓住梯子并慌张低头；新娘与宾客后退；镜头快速后撤并以白闪结束。保持同一人物、梯子、婚礼场地和灯架结构，不生成字幕或台词文字。

## 5. 字幕配置

视频文件保持无游戏字幕。字幕保存在 `W101_captions.json`：

```json
[
  {
    "start": 0,
    "end": 2.2,
    "text": "有梯子，新娘还是死了",
    "style": "result"
  },
  {
    "start": 2.2,
    "end": 5,
    "text": "他解决了身高，没有解决专业",
    "style": "explanation"
  },
  {
    "start": 5,
    "end": 8,
    "text": "够得到，不等于会修",
    "style": "comment"
  }
]
```

时间单位为秒，区间采用左闭右开。播放器后续应以视频 `currentTime` 计算当前字幕，不使用独立计时器。

## 6. 文件组织

```text
刷到你了/assets/video-tests/W101/
├── W101_keyframe_v1.png
├── W101_ltx_raw_v1.mp4
├── W101_thumbnail_v1.jpg
├── W101_captions.json
└── generation-notes.md
```

不生成烧录字幕版。`generation-notes.md` 记录提示词、任务 ID、实际消耗、时长、尺寸、生成次数与验收结果，不记录 API Key 或签名下载链接。

## 7. 失败处理

- 关键帧中梯子不完整、人物无法合理站立或手部严重变形：付费调用前重新编辑关键帧；
- 上传或鉴权失败：停止，不提交任务，不打印密钥；
- 任务失败：记录返回原因后，才可使用第二次额度；
- 视频没有明确的拧错动作、梯子消失或灯架完全不动：允许一次收紧提示词后的重试；
- 轻微运动模糊：保留，不因普通动态模糊重复扣费；
- 结果合格：直接生成缩略图、配置和记录，不做字幕烧录。

## 8. 验收标准

- 9:16 竖屏，时长不超过 10 秒；
- 第一眼能看见梯子、梯子上的伴郎和上方灯架；
- 五秒内能看懂伴郎拧错零件导致灯架失稳；
- 梯子在主要动作段持续存在，不凭空消失；
- 人物、梯子和灯架没有影响理解的持续严重变形；
- 视频有真实动作，不是静态推拉；
- 无血腥、直接撞击、乱码、品牌或水印；
- 视频本体无游戏字幕；
- `W101_captions.json` 格式正确、时间连续且不超过视频时长；
- API Key 未写入任何项目文件或 Git 历史。

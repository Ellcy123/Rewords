# 手机局域网测试规则设计

**日期：** 2026-07-20

**适用项目：**《刷到你了》Web 原型

**状态：** 已获方向确认，等待文档复核

## 目标

后续每个可玩阶段都优先让测试者在真实手机上操作，而不是只在电脑浏览器中模拟手机。电脑负责运行本地服务，手机通过同一局域网中的 IPv4 地址访问，以尽早发现触摸、纵向视频、移动端安全区和小屏布局问题。

## 默认测试方式

开发期间使用 Vite 开发服务器，并固定优先使用端口 `4173`。当前 Phase A 分支使用：

```powershell
cd "D:\Rewords\.worktrees\phase-a-deterministic-core\刷到你了\web-prototype"
npm run dev -- --host 0.0.0.0 --port 4173
```

分支合并并清理临时 worktree 后，标准目录改为 `D:\Rewords\刷到你了\web-prototype`。长期规则以“当前工作副本中的 `刷到你了\web-prototype`”为准，不依赖某个固定 worktree 名称。

测试者使用终端输出的 `Network` 地址，例如：

```text
http://192.168.1.23:4173/
```

如果 `4173` 已被占用，依次改用 `4174`、`4175`，并在测试记录中写明实际端口。不得仅记录示例 IP；每次都以本次启动输出为准。

## 网络条件

- 手机和电脑必须连接同一个普通 Wi-Fi 或同一个可互访的局域网。
- 不使用开启客户端隔离的访客 Wi-Fi。
- Windows 防火墙仅放行“专用网络”，不要求开放公共网络。
- 测试异常时，先关闭手机流量、VPN 和代理，再排查应用问题。
- 电脑重新连接 Wi-Fi 后 IPv4 地址可能改变，旧书签不保证继续有效。

## 两种运行层级

### 日常开发测试

使用 `npm run dev -- --host 0.0.0.0 --port 4173`。优点是修改后刷新手机即可看到，适合频繁迭代。

### 阶段验收测试

先完成类型检查、自动化测试与生产构建，再使用生产预览服务：

```powershell
npm run typecheck
npm test
npm run build
npm run preview -- --host 0.0.0.0 --port 4173
```

若项目尚未提供 `preview` 脚本，阶段验收前补充该脚本；在补充之前，可以继续使用开发服务器进行人工移动端测试，但必须在记录中注明。

## 每次测试的固定流程

1. Codex 先完成该阶段要求的自动化测试和构建检查。
2. Codex 启动局域网服务，向用户明确提供本次真实 `Network` 地址，不只说“访问电脑 IP”。
3. 用户使用真实手机浏览器打开地址；需要全新存档时优先使用无痕窗口，或清除该地址的网站数据。
4. 按当前阶段的人工测试脚本操作，记录入口、关键选择、余额、背包、消息和解锁结果。
5. 发现问题时同时记录手机型号、系统、浏览器、屏幕方向、测试提交、访问地址和复现步骤。
6. 测试完成后关闭本地服务，避免端口长期占用。

## 最低记录字段

每次手机人工测试至少记录：

- 测试日期与测试提交哈希；
- 手机型号、系统版本、浏览器；
- 电脑与手机的网络类型；
- 实际访问地址与端口；
- 是否使用全新存档；
- 测试路线及关键选择；
- PASS/FAIL 结果；
- 失败时的截图、录屏或明确复现步骤。

局域网 IP 属于临时环境信息，不作为账号密码或密钥处理，但发布测试记录时可以只保留网段形式，例如 `192.168.1.x:4173`。

## 长期协作规则

- 当用户说“开始手机测试”“给我测试地址”或表达同等含义时，Codex 默认采用本规则，不再反复询问测试方式。
- 默认优先真实手机同网测试；电脑响应式模拟仅作为补充，不替代真机测试。
- 开发测试默认端口为 `4173`，阶段验收也优先保持同一端口。
- 若同网访问失败，Codex应依次检查服务监听地址、实际 IPv4、Windows 专用网络防火墙、访客网络隔离、VPN/代理和端口占用。
- 只有手机与电脑无法处于同一网络时，才考虑临时公网隧道；公网暴露不是默认流程。

## 当前阶段边界

Phase A 手机上可以测试现有视频流、原有改命交互、小黄车、运营任务和经济补助。PK 二元选择、炎鑫私聊和主动消息界面属于 Phase B；Phase A 已完成其规则层，但不能把尚未实现的界面列入本轮手机验收。

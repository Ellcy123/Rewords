# 《刷到你了》试玩版合并与真人视频接入设计

> 日期：2026-07-18
>
> 状态：方案已确认

## 1. 目标

将 `feature/shuadaonile-web-prototype` 上已经完成的 React/Vite 试玩版正式合并到 `main`，并把现有 W001、W101 两条真人视频接入试玩版。完成后在本机启动可访问网址，使用浏览器验证推荐流播放、配置字幕、暂停和节点切换。

## 2. 分支处理

试玩版此前在隔离功能分支与临时 worktree 中开发。功能完成后没有执行合并，导致 `main` 只包含设计文档和后来生成的视频资产，而试玩代码仍停留在功能分支。

本次在 `main` 执行非压缩合并，保留试玩版原有提交历史。合并前确认：

- 当前 `main` 只保留用户已有的未跟踪 `.DS_Store`；
- 旧 worktree 路径已经不存在，仅留下可清理的 Git 注册记录；
- 功能分支代码、测试和 hosting 配置完整；
- 合并不删除 W001、W101 或其他当前主分支资产。

合并后继续在 `main` 完成媒体接入，不再让试玩版与正式资产分处两个分支。

## 3. 媒体配置

扩展 `VideoNode`，增加可选的媒体配置。配置至少包含：

- 视频文件 URL；
- 封面 URL；
- 字幕 cue 数组；
- 每条 cue 的 `start`、`end`、`text` 与 `style`。

W001、W101 使用无游戏字幕的真人原片。播放器字幕由视频 `currentTime` 选择当前 cue，不使用独立定时器，暂停、恢复和循环后不会漂移。

其他 14 个节点不配置视频，继续使用现有 CSS 动态分镜与 `StoryBeat`，保证原型完整流程不因只有两条正式视频而中断。

## 4. 资产组织

将浏览器直接加载的正式媒体放在：

```text
刷到你了/web-prototype/public/media/
├── W001_ltx_raw_v1.mp4
├── W001_thumbnail_v1.jpg
├── W101_ltx_raw_v1.mp4
└── W101_thumbnail_v1.jpg
```

源文件继续保留在 `刷到你了/assets/video-tests/`，不覆盖、不移动。`public/media` 使用副本，使 Vite 开发服务器和静态构建都能以稳定的 `/media/...` URL 加载。

W101 字幕沿用已验证的 JSON 时间轴。W001 将现有 ASS 字幕转换成同一 cue 结构，只保留游戏需要的文本与时间，不把 ASS 样式耦合进 React 数据模型。

## 5. 播放器行为

`StoryStage` 根据节点是否有视频媒体选择渲染方式：

- 有媒体：渲染 `<video playsInline loop>`、封面和配置字幕；
- 无媒体：保留现有 CSS 分镜、beat 轮播和频道装饰；
- 当前卡片激活且全局未暂停时播放；
- 卡片离开当前视口或打开抽屉时暂停；
- 点击舞台切换全局暂停状态；
- 浏览器拒绝自动播放时保持首帧并允许用户点击恢复；
- 视频加载失败时自动显示现有 CSS 分镜，不出现空白页。

视频本体声音遵循浏览器自动播放限制。首次测试重点是画面、字幕和操作，不新增独立声音设置逻辑。

## 6. 样式

真人视频使用 `object-fit: cover` 填满现有 9:16 舞台。字幕放在底部文案和右侧按钮之上的安全区，采用高对比描边或半透明底，避免遮挡新娘、梯子、商品卡和“改命礼物”。

W001 与 W101 不再同时显示旧的 `StoryBeat` 大字动画，防止字幕重复。账号、标题、互动按钮、商品卡和底部导航仍由现有短视频 UI 覆盖在视频上。

## 7. 测试与验收

自动测试覆盖：

- W001、W101 节点媒体 URL 和字幕配置合法；
- 每条 cue 满足 `start < end`、互不重叠且不超过节点时长；
- 有视频的节点渲染 `<video>` 与当前字幕；
- 无视频节点仍渲染现有 CSS 分镜；
- 播放失败时切回 CSS 分镜；
- 现有完整通关、推荐流、存档和商品测试继续通过。

运行 `npm test`、`npm run typecheck`、`npm run build`。随后启动 Vite 开发服务器，通过本地浏览器确认：

1. W001 真人视频能播放并显示配置字幕；
2. 上下滑动后其他节点正常；
3. 触发梯子改命后 W101 使用第二条真人视频；
4. 点击暂停、继续和循环时字幕同步；
5. 控制台没有资源 404 或 React 错误。

## 8. 交付

- 完整试玩版位于 `main`；
- W001/W101 真人视频已配置化接入；
- 本地测试网址在任务完成时保持运行并自动打开；
- 合并、实现与验证提交推送到 `origin/main`；
- 不提交现有 `.DS_Store`。

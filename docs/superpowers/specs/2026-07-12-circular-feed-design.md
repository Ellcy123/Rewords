# 推荐流循环轮播设计

## 目标

将当前有首尾边界的三卡推荐流改成双向无限循环。三条内容按 `1 → 2 → 3 → 1` 向后循环，也可按 `1 → 3 → 2 → 1` 向前循环。循环结算时必须保留已滑入卡片的 DOM 与分镜状态，不能重建或闪屏。

## 方案

采用“虚拟位置 + 实际索引取模”。推荐流维护一个可无限增减的虚拟位置，例如 `-1、0、1、2、3`；实际视频索引通过安全取模映射到现有节点。三张卡的 React 身份使用虚拟位置，因此下一槽滑入当前槽时仍是同一个元素。

- 多于一条视频时，触屏、滚轮和上下方向键均可双向循环。
- 只有一条视频时不发生翻页，小幅位移后回弹。
- 三卡窗口始终渲染 `虚拟位置 - 1 / 当前虚拟位置 / 虚拟位置 + 1`。
- 父级仍接收普通的 `0..count-1` 实际索引，游戏存档与剧情状态不记录虚拟位置。
- 礼物结算、线索跳转或推荐列表改变时，若当前实际节点不再匹配，则将虚拟位置安全对齐到父级索引。

## 状态与数据流

1. `VideoFeed` 保存本地虚拟位置，初值为父级实际索引。
2. `useFeedNavigation` 在循环模式下只判断 `count > 1`，不再对首尾施加阻尼或拒绝翻页。
3. 动画完成后，手势层返回新的虚拟位置。
4. `VideoFeed` 更新虚拟位置，并通过安全取模把实际索引交给 `FeedScreen`。
5. 三张卡使用虚拟位置作为 key；刚刚滑入的卡片不卸载，字幕拍点和合成层连续。

## 边界与测试

- 三条视频：验证最后一条向后进入第一条、第一条向前进入最后一条。
- 两条视频：上下两个邻居可以映射到同一节点，但虚拟位置不同，卡片身份不冲突。
- 一条视频：拖动回弹，不触发索引变化。
- 循环结算：验证滑入卡片元素在结算前后保持同一 DOM 身份。
- 保留现有距离阈值、速度阈值、点击抑制、锁定状态和 240ms 动画。

# Wrong Pivot —— 项目目标与规则基准

> 版本：v1.0 ｜ 日期：2026-07-03 ｜ GGJ 主题：Anchor（锚点）
> 本文档 = 立项目标 + 方案评估 + 全部规则裁定。开发以此为准。

---

## 一、一句话定位

**你不能移动任何东西，你只能决定"动作从哪里发生"。**

一个坏掉的关卡编辑器里，所有道具的动画都写死了：门会转、杆会伸、弹簧会弹。但它们的锚点全错了。玩家唯一的能力：**拖动锚点，按 Space 运行一拍。**

主题契合度是字面级的：Anchor = pivot = 锚点，不需要任何解释成本。

## 二、为什么这个方案成立（评估结论）

1. **单一动词设计。** 玩家全程只有一个操作（拖锚点 + 运行一拍），没有任何操作学习成本，所有深度都在"理解"层面——这是谜题游戏的黄金结构。
2. **谜题问的不是"放哪"，而是"这个动作换个圆心会变成什么"。** 门=桥=锤=电梯，同一段动画因锚点不同而改变身份。"啊？还能这样？"的密度天然就高。
3. **表现即教学。** 编辑器风格 + 发光十字锚点 + 虚线预览弧，视觉语言自解释，美术需求极低，Jam 友好。
4. **失败也好笑。** 锚点放错时道具会做出荒谬又合理的动作（风车门、自弹弹簧），失败本身就是内容。

## 三、方案原稿留下的 8 个空白（已裁定）

原方案没有回答以下问题，任何一个含糊都会让关卡无法设计。以下为裁定，理由附后：

| # | 问题 | 裁定 | 理由 |
|---|---|---|---|
| 1 | 按一拍时，是所有道具都动，还是只有选中的动？ | **所有道具同时执行各自的动画** | 副作用是戏剧性的来源：你为了 A 挪锚点，B 也会跟着动一下。配合撤销，制造"编排感" |
| 2 | 锚点是钉在世界上，还是跟着道具走？ | **物体空间（跟着道具走）**，与真实编辑器语义一致 | 支撑两个核心涌现玩法：同一锚点连按两拍=道具"翻滚行走"；弹簧反冲后锚点仍在背面=被钉死在墙角不再乱跑 |
| 3 | 旋转方向能改吗？ | **不能。永远顺时针 90°** | "动画是写死的"是本作的纯洁性。方向可选会让谜题深度减半（翻滚行走就是被这个限制逼出来的） |
| 4 | 动画撞墙怎么办？ | **动画中断，哐一声弹回原位（clank 规则）** | 宽容、有音效反馈、且可以拿来当关卡的"墙"用（见关卡设计中的天花板） |
| 5 | 道具受重力吗？ | **道具无重力（只有动画）；玩家和箱子有重力** | 板可以悬空当桥，规则一句话讲清 |
| 6 | 站在道具上的东西怎么办？ | **被一起带走，动画结束后落在道具新姿态的最高处（乘骑规则）** | 摩天轮玩法的前提 |
| 7 | 锚点位置连续还是吸附？ | **吸附到半格网格** | 谜题确定性、可提示、可验证。连续锚点会让解产生毫米级容错问题 |
| 8 | 预览怎么做？ | **常驻幽灵预览**：锚点放下的瞬间，直接显示这一拍运行后的半透明结果姿态 + 虚线圆弧 | 90° 任意圆心旋转的心算负担超过多数玩家。幽灵预览不是 UX 加分项，是本作能不能玩的生死线。**这是全项目最高优先级的单项。** |

## 四、操作定义

| 输入 | 效果 |
|---|---|
| 鼠标拖动发光十字 | 移动该道具的锚点（吸附半格，可放在空气、墙内、任何位置——锚点是几何点，不参与碰撞） |
| Space | 运行一拍（所有道具同时执行动画） |
| Z | 撤销一拍（含这拍前的锚点编辑） |
| R | 重开本关 |
| ←→ / 跳 | 玩家角色移动，1 格跳跃，2 格身高，无摔落伤害（尖刺除外） |

拍与拍之间可以编辑任意多个锚点——限制次数不会增加深度，只会增加烦躁。

## 五、MVP 范围（Jam 48h）

**三个道具，一个都不多：**

| 道具 | 写死的动画 | 锚点改变的身份 |
|---|---|---|
| 旋转板（3 格长） | 顺时针转 90° | 门 / 桥 / 扫臂 / 翻滚行走 / 远心摩天轮 |
| 弹簧块（1 格） | 向锚点反方向弹出冲击 | 弹玩家 / 弹箱子 / **反冲弹自己** |
| 伸缩杆（2↔4 格） | 长度翻倍↔缩回（逐拍切换） | 搭桥 / 顶箱子 / 卡陷阱 / 外锚点=伸长时整体位移 |

> ⚠ 伸缩杆的"外锚点=缩放即位移"数学上很美但视觉上最难读，**排期放最后，砍它不砍前两个**。

**关卡量：6~7 关。** 前 5 关每关只教一个锚点姿势（对应原方案第 1~5 关），第 6 关双道具联动，第 7 关为毕业考（见《01_关卡设计_圆心在别处.md》）。

**美术：** 纯编辑器风——网格背景、线框道具、发光十字、虚线弧。故障感用色差抖动 shader 一个就够。

## 六、成功标准

- 每一关必须有至少一个可指认的"啊？还能这样？"瞬间，说不出来的关卡砍掉；
- 玩家在不读任何文字的情况下通过第 1 关（幽灵预览承担全部教学）；
- 通关平均时长 15~20 分钟，符合 Jam 评委耐心预算。

## 七、风险清单

| 风险 | 对策 |
|---|---|
| 90° 心算劝退 | 幽灵预览常驻，第一优先级开发 |
| "所有道具每拍都动"导致混乱 | 关卡内道具总数 ≤3；撤销无限；无关道具的动画设计成原地无害（如弹簧默认朝天弹） |
| 玩家用蛮力穷举锚点位置 | 吸附网格让穷举空间可控，但真正的防线是关卡设计：解必须依赖"换一种理解"，而非"换一个格子"（见 01 文档的诱饵设计） |
| 乘骑物理写不稳 | 乘骑规则简化为"落在新姿态最高处"，不做真实旋转跟随 |

# 关卡设计规则

本文档定义《词语炼金》每个关卡节点的设计流程与约束。所有关卡设计必须遵循此流程。

## 1. 设计流程

关卡设计严格按以下顺序进行，不可倒置：

```
分支剧情 → 结果卡牌 → 场景物品
```

### 第一步：设计分支剧情

确定当前节点的故事情境和目标，然后设计该节点所有可能的剧情分支。每个分支是一个完整的叙事走向，包含：

- 分支名称
- 触发后的结果状态或剧情方向
- 对后续节点的影响
- 数值变化

分支数量：每个节点 3–5 个分支。

**分支剧情只定义结果，不描述具体执行过程。**

关卡设计阶段不写死玩家或角色“如何做到”的细节过程，例如“把花缝到衣服上”“用口红在裙子上画纹样”等。这些属于后续表现形式、交互动画、文本演出或美术呈现，不应在分支规则中绑定。

分支剧情应关注：

- 玩家最终获得了什么结果卡牌
- 角色当前处于什么状态
- 该结果如何改变当前节点走向
- 该结果如何影响后续节点
- 数值或标签发生了什么变化

也就是说，分支描述的是“结果是什么”，不是“过程怎么演”。

### 第二步：设计结果卡牌

每个分支对应一张结果卡牌。结果卡牌是玩家通过拆字合成制造出来的新卡牌，持有它即触发对应分支。

每个节点固定 **4 张结果卡牌**。

每张结果卡牌包含：
- 卡牌名称
- 卡面图像
- 合成配方（由哪些字组成）
- 对应的剧情分支
- 数值效果

**结果卡牌不存在"辅助"或"工具"类型。** 每张结果卡牌都必须指向一个剧情分支，都与叙事直接相关。

### 第三步：推导场景物品

根据 4 张结果卡牌的配方，反推场景中需要哪些物品。场景物品的选择标准：

1. 物品的可拆字中包含结果卡牌配方所需的字
2. 物品在当前场景中自然存在（房间里有什么就放什么，不硬凑）
3. 每张物品卡拆出的字中，有用的字构成收益，无法参与任何合成的字构成代价（剩余词）

场景物品数量：每个节点 5–7 张。

## 2. 核心约束

### 结果卡牌数量

每个节点固定 4 张结果卡牌。

### 场景卡牌获取

玩家获得当前场景中**全部**陈列的场景卡牌，没有选择限制。所有场景卡牌都会被拆解，全部字进入可用字池。

### 玩家合成限制

玩家从可用字池中合成结果卡牌，数量限制：

- 最少：1 张
- 最多：2 张

这是硬性规则限制，不是字数自然约束。玩家手中可能有足够的字合成 3–4 张结果卡，但规则只允许选择其中 1–2 张。未被合成消耗的字全部进入剩余词处理。

玩家的核心决策变为：**我有这些字，我选择合成哪 1–2 张结果卡？** 选择不同的结果卡组合，触发不同的剧情分支，同时产生不同的剩余字和对应代价。

### 配方独占

合成配方是策划预设的固定列表。不在配方表中的字组合无法合成任何结果卡牌。每个字在一次合成中只能使用一次（用过即消耗）。

### 场景物品固定

场景中陈列的物品是固定的，由策划预设。玩家获得全部场景卡牌并拆解，不存在"选哪些场景卡"的环节。

## 3. 卡牌结构

### 场景物品卡

```
名称：花瓶
图像：窗台上插着野花的瓷瓶
可拆字：花 / 瓶
类型：物品
```

场景物品卡可以是：
- 物品（花瓶、旧裙子、口红……）
- 状态/情感（心动、害怕……）

场景物品卡可以是 2 字或 3 字。3 字卡拆出更多字，给玩家更多合成空间，但也产生更多潜在剩余。

### 结果卡牌

```
名称：花裙
图像：旧裙子上缀满了野花
配方：花（来源：花瓶）+ 裙（来源：旧裙子）
分支：华服路线——她穿着花裙从正门入场
效果：体面 +1
```

每张结果卡牌必须：
1. 有明确的合成配方
2. 指向一个具体的剧情分支
3. 有独立的卡面图像和叙事描述

## 4. 剩余词规则

玩家拆解场景物品后，未被合成消耗的字成为"剩余字"。

剩余字不能丢弃，必须使用基础组句词（我 / 你 / 在 / 了）组成句子，由 AI 评分。

评分维度：
- 合理分：句子是否说得通
- 抽象分：是否荒诞或有节目效果
- 诗意分：是否有意象
- 负担分：是否造成负面后果

剩余字越多、组句越不通顺，惩罚越重。

这意味着：每张场景物品卡中无法参与合成的"废字"就是选择该卡的代价。玩家拿走有用的字，就必须承担没用的字。

## 5. 设计检查清单

完成一个节点的设计后，用以下问题自检：

- [ ] 是否从剧情分支开始设计，而不是从场景物品开始？
- [ ] 结果卡牌是否恰好 4 张？
- [ ] 每张结果卡牌是否都指向一个独立的剧情分支？
- [ ] 是否存在"辅助卡"或"工具卡"？（不应该存在）
- [ ] 场景物品是否在该场景中自然存在？
- [ ] 玩家拿到全部场景卡后，拥有的字是否足够合成多张结果卡（但规则限制只能选 1–2 张）？
- [ ] 合成 2 张结果卡的组合是否存在？是否设计了对应的组合分支？
- [ ] 选择不同的 1–2 张结果卡时，剩余字数量和处理难度是否有差异（选得多→剩得少→代价小，但分支可能不同）？
- [ ] 不在配方表中的字组合是否确实无法合成？

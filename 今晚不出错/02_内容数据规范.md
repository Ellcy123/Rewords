# 02 内容数据规范

面向：策划填表 / 程序解析。所有内容以 JSON 驱动，程序不写死任何稿件文本。**本作 80% 的工作量在这份数据上，结构要一次定对。**

---

## 1. 数据文件总览

```
data/
├─ script.json        # 三幕稿件（线性剧本，含所有句子与错误标注）
├─ rules.json         # 台规定义与生效时点
├─ cards.json         # 赞助卡 / 插播稿
├─ danmaku.json       # 弹幕语料池
└─ titles.json        # 结算称号表
```

## 2. 稿件与错误标注（script.json）

稿件按 幕 → 段落 → 句子 → 词块 四级组织。**中文预分词由策划在表里手工完成**（词块粒度=可点击单元，通常 1–4 字），不做运行时分词。

```jsonc
{
  "acts": [
    {
      "id": "act1",
      "cursorSpeed": 3.0,            // 字/秒
      "paragraphs": [
        {
          "id": "a1p2",
          "gapAfter": true,           // 段后是否有可拖卡间隙
          "sentences": [
            {
              "id": "a1p2s1",
              "tokens": [
                {"t": "1943年"},      // 普通词块只有文本
                {"t": "，"},
                {"t": "哥伦布"},
                {"t": "的"},
                {"t": "船队"},
                {"t": "到达"},
                {"t": "美洲"},
                {"t": "。"}
              ],
              "errors": [
                {
                  "tokenIndex": 0,          // 错在哪个词块
                  "type": "history",        // 错误类型（决定弹幕池）
                  "correct": "1492年",
                  "distractors": ["1592年", "1942年"],  // 干扰项
                  "difficulty": 1           // 1常识 2进阶 3刁钻（配平用）
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

### 关键约定

- `errors` 为空数组 = 干净句。
- **陷阱句**（两种改法都自洽）：`errors` 仍只标一个真错误，但把"另一种自洽改法"放进 `distractors`，例：
  - 句子："位于长江入海口的城市天津……"
  - 真错误标在"长江"（correct: "海河"），distractors 含 "天津→上海" 方向的干扰表述。策划备注写明考点。
- 台规生效后的"禁字/替换"命中，**不在 script.json 里重复标注**，由 rules.json 的 `hitTokens` 指向（见 §3），避免双份维护。
- 每句字数 ≤ 25，保证同屏完整可见。

### 错误类型枚举（type）

`history` 历史 / `geo` 地理 / `science` 科学 / `common` 生活常识 / `pop` 流行文化 / `logic` 句内自相矛盾 / `rule` 台规命中（保留值，运行时生成）

### 内容量基准（MVP）

| 幕 | 段落 | 句子 | 埋错数 | difficulty 配比 |
|---|---|---|---|---|
| 一幕 | 4 | ~14 | 8 | 全 1 |
| 二幕 | 5 | ~18 | 14 | 1:2 = 60:40 |
| 三幕 | 5 | ~18 | 12（+台规命中约10处） | 1:2:3 = 40:40:20 |

## 3. 台规（rules.json）

```jsonc
{
  "rules": [
    {
      "id": "rule_sponsor",
      "type": "duty",                     // ban 禁字 | duty 义务 | replace 替换
      "announceAt": "a2p1",               // 在哪个段落开始前宣布（耳机气泡）
      "title": "赞助商口播",
      "announce": "导播：接了个大单！蓝天矿泉水，每段播报口播一次，词马上发你！",
      "icon": "sponsor",
      "duty": {"cardId": "card_sponsor_1", "intervalSec": 60, "timeoutSec": 30}
    },
    {
      "id": "rule_ban4",
      "type": "ban",
      "announceAt": "a2p3",
      "title": "禁字：4",
      "announce": "导播：台里刚下的通知，'4'不吉利，一个都不许念！",
      "icon": "ban4",
      "hitTokens": [                       // 策划预枚举所有命中位置
        {
          "sentenceId": "a2p4s2",
          "tokenIndex": 3,                 // "1944年"
          "correct": "上世纪四十年代中期",
          "distractors": ["1九44年", "一九四四年"]   // 干扰项本身仍含"四"读音陷阱
        }
      ]
    },
    {
      "id": "rule_replace_baiyun",
      "type": "replace",
      "announceAt": "a3p1",
      "title": "替换：白云",
      "announce": "导播：竞品白云矿泉水出事了！所有'白云'给我换掉，天气预报也不例外！",
      "icon": "replace",
      "hitTokens": [
        {
          "sentenceId": "a3p2s3",
          "tokenIndex": 5,                 // 天气预报里的"白云"
          "correct": "那种白色的云状物",
          "distractors": ["白色的云", "不可描述的云"]
        }
      ]
    }
  ]
}
```

### 关键约定

- **无运行时规则引擎**：三幕是线性剧本，每条规则生效后会命中哪些词块，策划全部预枚举在 `hitTokens`。程序只做两件事：到 `announceAt` 时弹气泡+上台规栏；把 `hitTokens` 注入对应句子的错误列表（type=rule）。
- **规则冲突**同样预写：冲突体现在后续 hitTokens 的 correct/distractors 里（如赞助口播卡 v2 的词里含"4"，其 hitToken 的正确项是"肆"写法）。
- 简单模式（⚙ 全局开关）：`hitTokens` 命中的词块渲染微弱下划线。

## 4. 卡片（cards.json）

```jsonc
{
  "cards": [
    {
      "id": "card_sponsor_1",
      "type": "sponsor",
      "label": "蓝天矿泉水口播",
      "readText": "本节目由蓝天矿泉水独家冠名，蓝天矿泉水，喝了不上头。",
      "timeoutSec": 30,
      "rewards": {"onTime": 4, "timeout": -8, "midSentence": -5}
    },
    {
      "id": "card_breaking_1",
      "type": "breaking",
      "label": "突发：体育快讯",
      "timeoutSec": 20,
      "paragraphRef": "a3p_breaking1"     // 指向 script.json 中的插播段（内含埋错）
    }
  ]
}
```

- `midSentence`（拖进句中）额外效果：口播文本以红色词块插入当前句并被念出，判事故 −5，同时触发弹幕 `sponsor_fail` 池（狂欢向，不是纯嘲讽——这是"事故但好笑"的正反馈）。

## 5. 弹幕语料（danmaku.json)

```jsonc
{
  "pools": {
    "idle":        ["主播今天状态不错", "来了来了", "前排"],
    "err_history": ["义务教育漏网之鱼", "1943？穿越了？", "历史老师连夜辞职"],
    "err_geo":     ["地理课代表气晕", "建议重修初中地理"],
    "err_fixwrong":["他知道错了但他不知道对的", "纠错纠了个寂寞"],
    "misclick":    ["复读机成精了", "卡带了卡带了"],
    "water":       ["主播是骆驼吗", "这节目改名叫今晚喝不完吧", "第{n}杯了兄弟们"],
    "redzone":     ["导播脸都绿了", "收视率保卫战", "要完要完"],
    "sponsor_fail":["蓝天矿泉水三分球笑死", "无缝植入（物理）"]
  },
  "burst": {"onError": [8, 12], "normalPerSec": 0.8}
}
```

- 弹幕支持 `{n}` 占位符（喝水次数等）。
- **红线：任何池不得包含正确答案或答案暗示。**

## 6. 称号（titles.json）

```jsonc
{
  "titles": [
    {"id": "cctv",    "name": "央视预备役",        "cond": "fixRate>=0.9"},
    {"id": "fish",    "name": "义务教育漏网之鱼",   "cond": "commonErrors>=5"},
    {"id": "camel",   "name": "喝水冠军",          "cond": "waterCount>=8"},
    {"id": "sponsor", "name": "甲方孝子",          "cond": "sponsorOnTimeRate==1"},
    {"id": "stitch",  "name": "缝合怪",            "cond": "fixWrongCount>=3"},
    {"id": "cut",     "name": "因技术原因",        "cond": "rating==0"}
  ],
  "priority": ["cut", "cctv", "fish", "stitch", "camel", "sponsor"]  // 只授最高优先一个
}
```

- `cond` 由程序以简单表达式求值（字段全部来自结算统计对象），不做通用脚本。

## 7. 埋错写作规范（策划自查清单）

1. **常识错误要"错得响亮"**：一幕的错必须让 90% 观众瞬间反应过来（哥伦布 1943），观众抢答感是传播核心。
2. **进阶错误要"查得到、猜得着"**：二幕允许"我好像知道"级（活字印刷 vs 雕版的朝代），三选一的正确项要能靠排除法逼近。
3. **刁钻错误宁缺毋滥**：三幕每段至多 1 个 difficulty=3，且必须配一个"荒诞但明显"的干扰项保底笑点。
4. **陷阱句必须双向自洽**：改 A 或改 B 都能让句子成立，考的是"哪个更可能是稿子本意"，策划备注写清判定理由。
5. **禁字/替换的命中要有一处"荒诞命中"**：规则波及到不该波及的地方（天气预报的白云）是本作最重要的笑点类型，每条 ban/replace 规则至少预埋一处。
6. 所有事实错误的"correct"值需两人交叉核实——**纠错游戏自己出错是公关灾难**。

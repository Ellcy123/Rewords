# 01 核心玩法与系统设计

面向：程序 / 策划 / 美术共用。所有数值为首版基准值，标注 ⚙ 的项开放调参。

---

## 1. 屏幕布局

```
┌─────────────────────────────────────────────────────┐
│  ① 演播室画面（主播半身像 + 台标 + 新闻背景图）        │
│     ├ 主播有状态动画：正常念稿/卡壳/喝水/咳嗽/崩溃      │
│     └ ④ 屏幕内弹幕层（从画面上飘过，模拟直播间）        │
│  ② 收视率条（顶部，实时曲线 + 当前数值）               │
├─────────────────────────────────────────────────────┤
│  ③ 提词器区（屏幕下半 40%，当前段落全文可见）           │
│     ├ 朗读光标：卡拉OK式逐字高亮，匀速推进              │
│     └ 玩家可点击任意"词块"                             │
├─────────────────────────────────────────────────────┤
│  ⑤ 手牌区（赞助卡/插播稿，横排 0–3 张）                 │
│  ⑥ 台规栏（已生效规则的图标列表，悬停看详情）           │
│  ⑦ 资源区（水杯 ×N / 咳嗽 ×N）                         │
└─────────────────────────────────────────────────────┘
```

- 提词器按**词块（token）**渲染，不按字符。中文稿件在数据层预分词（见 02 文档），每个词块是一个可点击单元。
- 导播指令以"耳机气泡"形式从①区弹出，停留 4s，同时写入⑥台规栏。

## 2. 朗读光标（核心时钟）

- 光标逐词块推进，速度以"字/秒"计：
  - 第一幕 ⚙ 3.0 字/秒（约正常播音语速的 70%）
  - 第二幕 ⚙ 3.5 字/秒
  - 第三幕 ⚙ 4.0 字/秒，且突发插播时短暂 +0.5
- 光标**只在以下情况暂停**：喝水（3s）、玩家点中真错误后的卡壳窗（见 §3）、段落间隙（1.5s，用于拖卡）。
- 光标扫过未修正的错误词块 = **播出事故**（见 §6 判定）。

## 3. 交互一：点词 → 三选一（找错 + 改对）

**二段式是本作的知识考点所在：找错考"察觉不对劲"，改对考"真的知道"。**

1. 玩家点击任意词块：
   - **点中真错误**：主播进入"卡壳"状态（"呃——"音效），光标暂停 ⚙ 2.5s，弹出三选一浮窗（贴着词块显示，选项含 1 正确 + 2 干扰，干扰项来自数据表，见 02）。
     - 选对：词块替换为绿色闪烁一次，主播流畅接上。收视率 +2。
     - 选错：词块替换成另一个错误答案，光标继续，扫过时按播出事故判定，且弹幕触发专属嘲讽池（"他知道错了但他不知道对的"）。
     - 超时未选（2.5s 内没点选项）：视为选错处理，主播念出原始错误。
   - **点中正常词块（误点）**：主播无意义重读该词一遍（光标回退一个词块再前进），收视率 −1，弹幕触发"复读机"池。防乱点扫雷。
2. 光标已扫过的词块不可再点（生米煮成熟饭）。
3. 一个句子内可以有 0–2 个错误；同屏未处理错误上限 ⚙ 3 个（内容排布保证，不是运行时逻辑）。

## 4. 交互二：拖卡（插入类任务）

手牌区的卡片需拖入提词器的**段落间隙**（间隙在提词器上渲染为细缝，拖拽时高亮可放置区）：

| 卡类型 | 来源 | 时限 | 正确操作 | 失败后果 |
|---|---|---|---|---|
| 赞助口播卡 | 台规触发，周期性发放 | ⚙ 30s 倒计时 | 拖入任意段落间隙，主播完整念一遍口播词 | 超时：收视率 −8，导播咆哮；拖进句子中间：口播词嵌进新闻里念出（"詹姆斯投进了蓝天矿泉水三分球"），按播出事故 −5 但弹幕狂欢 +梗值 |
| 突发插播稿 | 剧本定点触发 | ⚙ 20s | 拖入间隙，当前稿子播完该段后切入插播稿（插播稿本身也埋错） | 超时：导播强制插入当前位置，打断正在处理的句子，光标直接跳转 |

- 拖卡与点词共用一只鼠标——**手牌积压时的调度压力是第二三幕的主要难度来源**，不另加操作复杂度。

## 5. 交互三：资源（刹车）

| 资源 | 按键 | 效果 | 数量 | 惩罚曲线 |
|---|---|---|---|---|
| 喝水 | 空格 | 光标暂停 ⚙ 3s，主播端杯动画 | 无限次 | 第 1–2 次免费；第 3 次起每次收视率 ⚙ −2、−3、−4 递增，弹幕触发"骆驼"池 |
| 咳嗽 | C 键或点图标 | 跳过当前整句（句内错误不判事故，内容也不计分） | 每局 ⚙ 2 次 | 每次收视率 −3，留给"完全不会"的死局 |

- 喝水在规则说明里明示"喝多了观众会烦"，把风险交给玩家自己权衡。

## 6. 收视率（生命值 + 计分合一）

- 范围 0–100，开局 ⚙ 50。
- 变动来源汇总：

| 事件 | 变化 |
|---|---|
| 正确修正一个错误 | +2 |
| 一整句无事故播完 | +1 |
| 播出事故（念出错误） | ⚙ −5 |
| 改错（三选一选错并念出） | −5，且弹幕加倍 |
| 误点重读 | −1 |
| 违反生效中的台规（见 §7） | 每次 ⚙ −4 |
| 赞助口播成功 | +4 |
| 赞助超时 | −8 |
| 喝水（第3次起）/咳嗽 | 递增 −2..−4 / −3 |

- 收视率 ≤ 20 时进入"红温"状态：画面边缘泛红、弹幕滚速加倍、导播喘气声——**先给崩溃演出，再给死亡**。
- 收视率 = 0：直播掐断，进结算。

## 7. 台规系统（规则只叠加不撤销）

台规 = 一条持续生效的判定规则，通过导播耳机气泡宣布，写入台规栏。**台规作用于"即将被念出的词块"**，由内容层预标注 + 运行时简单匹配实现（见 02 文档 tag 机制）。

### 台规的三种类型

1. **禁字型**：某字/词从现在起不能念出。稿件中命中的词块会在生效后**自动变为可点状态**（但不高亮提示——发现它也是玩家的活。⚙ 简单模式下给微弱下划线提示）。点中后三选一给出替代说法。
   - 例：「今日起数字'4'不吉利」→"1944 年"必须改选"一九四四年"的替代表述卡（"上世纪四十年代中期"）。
2. **义务型**：周期性必须做某事。
   - 例：「每段播报必须口播一次赞助商」→ 触发赞助卡发放循环。
3. **替换型**：特定词临场换成指定词。
   - 例：「竞品'白云矿泉水'出事了，所有'白云'临场替换」→ 天气预报里的"白云"也命中，三选一里出现"那种白色的云状物"这类荒诞正确项。

### 冲突设计（第三幕核心笑点）

台规之间要**预埋冲突**，冲突解法藏在三选一的选项里：

- 例：赞助商改名为"蓝天 4 号泉"，而"4"是禁字 → 口播词里的"4"必须再替换，正确项是"蓝天肆号泉"。
- 冲突不需要运行时规则引擎，**全部在内容层预写**：每条稿件按"当前生效台规集合"预置对应的错误标注与选项（三幕流程是线性剧本，生效集合可枚举）。

## 8. 三幕节奏表

| 幕 | 时长 | 光标速度 | 新增台规 | 错误密度 | 教学目标 |
|---|---|---|---|---|---|
| 一幕·开播 | ~3 min | 3.0 | 0 条（纯纠错） | 每句 ≤1，全是常识级（1943哥伦布） | 学会点词+三选一+喝水 |
| 二幕·加规 | ~4 min | 3.5 | +3 条（赞助口播→禁字→替换），间隔≥45s | 每句 0–2，出现"两种改法都自洽"的陷阱句 | 学会拖卡、台规栏、咳嗽 |
| 三幕·失控 | ~4 min | 4.0 | +2 条（与旧规冲突），突发插播 ×2 | 密度不再升，**认知负荷靠台规叠加** | 手忙脚乱的高潮体验 |

- 设计红线：第三幕的难不来自更快更密，来自"同时想着 5 条规则"。如果测试者反馈"手跟不上"而不是"脑子跟不上"，降光标速度，不降规则数。

## 9. 弹幕系统（表现层，不参与判定)

- 屏幕①区持续飘过弹幕，从**语料池**按当前状态抽取：
  - 常态池（"主播好帅""今天播啥"）
  - 事故池（按错误类型分池："义务教育漏网之鱼""1943？穿越了？"）
  - 改错池、复读池、骆驼池（喝水）、红温池
- 玩家**每犯一次错，弹幕即时爆一波**（瞬时 8–12 条）——这是"观众比玩家先看到"的具象化。
- 弹幕永远不给正确答案，只嘲讽和起哄（避免变成提示系统）。

## 10. 结算：今日播报事故集锦

- 记录本局所有事故事件（时间戳 + 事故句 + 当时弹幕最高赞）。
- 结算页逐条回放：显示原句、你念出的版本、正确版本、配当时弹幕。
- 三维评分：收视率均值 / 纠错率（改对数÷总错误数）/ 社死指数（事故数）。
- 称号表（按组合条件）：央视预备役（纠错率≥90%）、义务教育漏网之鱼（常识错误≥5）、喝水冠军（喝水≥8）、甲方孝子（赞助全部准时）、缝合怪（改错≥3）等。
- 提供"复制战绩"按钮（文本形式，含称号+数据），服务传播。

## 11. 真人主播模式（P2 彩蛋，最后有余力再做）

- 开关式模式：主播不出声，玩家对麦克风念稿。
- **只做音量检测，不做语音识别**：连续静默 >2s = 播出事故（吃螺丝）。
- 其余判定与普通模式一致。定位：UP 主录节目专用放大器，普通玩家可完全无视。

# 《今晚不出错》立项概述

- 版本：v0.1（GGJ 立项稿）
- 日期：2026-07-03
- 类型：单场景 / 文字纠错 + 资源调度 / 喜剧模拟
- 平台：Web（浏览器，桌面优先，兼容触屏）
- 单局时长：8–12 分钟（三幕结构）
- 目标场景：Game Jam 48 小时命题「Anchor」

---

## 1. 一句话概念

你是一名正在直播的新闻主播（Anchor），提词器上的稿子埋满事实错误，台里的规定还在不断增加——主播是自动念稿的，你是他脑子里那个负责纠错的声音。**全场都在崩，你是唯一不能崩的人。**

## 2. 主题契合（Anchor 三重解）

1. **News Anchor**：玩家角色即新闻主播，题材直接命中命题，且是绝大多数参赛队不会想到的义项（大多数队会做船锚/固定点机制）。
2. **主播双关**：中文语境下"主播"同时指 UP 主/直播博主，玩家群体（B 站用户）玩这个游戏时是"主播在扮演主播"，节目效果套娃。
3. **定海神针**：机制层面，玩家是整场直播中唯一的稳定器——规则在叠加、稿子在滚动、突发在插播，玩家的职责就是"锚住"这场直播不翻车。

## 3. 设计支柱（所有取舍以此为准）

| 支柱 | 含义 | 反例（不要做的） |
|---|---|---|
| 知识折磨，不是操作折磨 | 惩罚来自"不知道/没察觉"，不来自手速 | 不做需要精确点击/连打的操作 |
| 观众比玩家先看到 | 提词器全文可见，弹幕观众能抢答 | 不做隐藏信息、翻牌式题目 |
| 规则只叠加不撤销 | 新规则永不失效，与旧习惯冲突 | 不做"本关规则"式的清空重来 |
| 崩溃必须可视化 | 失败呈现为一场眼睁睁失控的直播 | 不做弹窗式"你失败了" |
| 语音是彩蛋不是门槛 | 默认全鼠标可玩，麦克风模式自愿开启 | 不做强制语音识别 |

## 4. 传播模型（自媒体友好性论证）

- **低门槛入口**：所有人都看过新闻播报，一秒看懂目标——"别让他把错的念出去"。
- **共鸣痛点**：念错字社死、直播翻车、甲方临时改需求，是中文互联网的集体创伤。
- **规则逐渐荒诞**：台规从"口播赞助商"叠到"禁字""纪年换算""竞品名临场替换"，每条新规都是一个视频爆点。
- **可视化崩溃**：实时收视率曲线 + 屏幕内弹幕嘲讽 + 导播耳机咆哮 + 结算"播出事故集锦"回放。
- **我上我也行**：观众永远比玩家先看到提词器上的错误，抢答欲拉满；结算集锦是天然的截图/切片传播素材。

## 5. 核心循环（30 秒版）

```
提词器滚动 → 朗读光标逼近 → 玩家扫描找错（点词）
→ 二段确认改对（三选一）→ 主播流畅播出 / 播出事故
→ 收视率涨跌 + 弹幕反馈 → 新台规到达 → 认知负荷升级 → 循环
```

穿插：拖入赞助口播卡 / 处理突发插播 / 喝水暂停 / 咳嗽跳句。

## 6. 胜负与结算

- **失败**：收视率归零 → 直播被掐断（黑屏 + "技术原因" 垫片），进入结算。
- **通关**：撑过三幕（约 10 分钟）完成当日播报，进入结算。
- **结算（无论胜负都要爽）**："今日播报事故集锦"——把本局所有念出口的错误、卡壳、喝水名场面剪成回放，配弹幕高亮 + 评分（收视率均值 / 纠错率 / 社死次数），给出称号（如"央视预备役""义务教育漏网之鱼""喝水冠军"）。

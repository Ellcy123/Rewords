# 项目协作核心规则

## 默认自主推进

- 对意图明确、风险低、可回滚且属于当前任务范围的细节，Codex 应结合现有产品与代码自行选择合理方案并直接实施，不要反复要求用户确认。
- 当存在多种实现方式，但它们不会实质改变玩法目标、工作范围、成本或数据安全时，默认采用最简单、最稳妥、最符合现有结构的方案；必要时在进度说明中简短写明假设即可。
- 用户说“开始”“继续”“可以”“好”或表达同等授权后，应连续完成设计取舍、实现、测试与当前项目既有的发布流程，不设置额外的审阅或逐项批准关卡。
- 只有在选择会明显改变核心玩法或需求范围、操作不可逆、可能丢失数据、需要新的外部权限或缺少无法安全推断的关键信息时，才向用户提问。
- 提问时一次只问真正阻塞推进的关键问题；能够通过检查项目、测试或采取安全默认值解决的问题，不交给用户反复选择。

## 手机局域网测试

- 《刷到你了》每个可玩阶段默认优先使用真实手机与开发电脑同一局域网测试，电脑响应式视口只作为补充。
- 开发服务默认从当前工作副本的 `刷到你了/web-prototype` 启动：`npm run dev -- --host 0.0.0.0 --port 4173`；若端口占用再依次使用 `4174`、`4175`。
- 每次向用户提供终端本次实际输出的 Network 地址，不复用旧 IP；阶段验收先运行类型检查、测试和构建。
- 同网访问失败时依次检查监听地址、电脑当前 IPv4、手机与电脑是否同一普通 Wi-Fi、Windows 专用网络防火墙、访客网络隔离、VPN/代理和端口占用。
- 测试步骤与记录模板统一维护在 `刷到你了/qa/mobile-lan-testing.md`。
